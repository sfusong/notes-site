# Development Track Course Roadmap in English

This note explains the most effective review order for the development-track materials if your goal is interview readiness.

## 1. Core idea

The right order is not:

- random technical topics first
- then projects
- then role fit

The better order is:

1. role positioning  
2. core project stories  
3. delivery and runtime concepts  
4. backend technical skeleton  
5. defensive fundamentals  
6. business and English polish

## 2. Why this order works

Your strongest value is not “I can recite Java theory deeply.”  
Your strongest value is:

- real fintech system exposure
- asset-management context
- delivery and integration experience
- rule-driven system experience
- enough technical depth to support believable project explanations

So project credibility should come first, and pure fundamentals should support it rather than replace it.

## 3. Suggested preparation stages

### Stage 0. Role positioning

Answer these questions first:

- what kind of developer role are you targeting?
- why are you a fit?
- what is your strongest story?
- where should you be careful not to overclaim?

Recommended note:

- [JD Fit and Candidate Positioning](notes/面试学习/开发岗/开发课程/01_岗位与计划/金融机构开发岗_JD能力拆解_English.md)

### Stage 1. Core project line

Before technical theory, make sure you can clearly explain:

- product lifecycle management system
- TA pre-batch warning and audit platform
- TA automation testing
- interface integration, UAT and rollout

### Stage 2. System runtime and delivery chain

Review how systems are built, released and connected:

- CI/CD
- Git / SDLC
- Tomcat / Nginx
- reverse proxy
- deployment concepts

Recommended note:

- [System and Delivery English Interview Pack](notes/面试学习/开发岗/开发课程/02_系统与交付基础/系统与交付基础_English_Interview_Pack.md)

### Stage 3. Backend technical skeleton

Review only what helps you explain real projects better:

- Spring Boot
- REST API
- request flow
- controller / service / repository layering
- validation
- SQL and schema evolution

Recommended note:

- [Backend Fundamentals English Interview Pack](notes/面试学习/开发岗/开发课程/04_语言与数据基础/后端基础_English_Interview_Pack.md)

### Stage 4. Defensive fundamentals

Cover the basics so you do not lose easy points:

- Java basics
- MySQL / Redis basics
- Linux usage awareness
- Windows Server awareness
- IaC awareness
- light algorithm review

### Stage 5. Business and English polish

Finish with:

- financial-domain understanding
- role-specific English phrasing
- concise self-introduction

Recommended notes:

- [Financial Domain English Interview Pack](notes/面试学习/开发岗/开发课程/05_岗位定制补充/金融业务认知_English_Interview_Pack.md)
- [Financial Developer English Expression](notes/面试学习/开发岗/开发课程/05_岗位定制补充/金融开发岗英语项目表达.md)

## 4. One-line summary

`Use projects to establish credibility, use system and backend fundamentals to support depth, and use business plus English polish to close strongly.`
