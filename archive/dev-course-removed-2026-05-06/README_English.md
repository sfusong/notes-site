# Development Track English Interview Guide

This page is the English-speaking entry for the development-track materials.  
It is not a literal translation of the whole knowledge base. Instead, it points you to the English notes that are most useful for real interviews.

## 1. Two main lines

### 1. Project interview pack

- [Project Interview Pack](notes/面试学习/开发岗/项目面试资料包/README.md)

Use this when you need to explain:

- your actual project experience
- system architecture
- business value
- your role boundaries

### 2. Non-project English interview pack

- [Development Course English Guide](notes/面试学习/开发岗/开发课程/README_English.md)

Use this when you need to explain:

- role fit
- SDLC / CI/CD / release flow
- backend basics
- business-domain understanding

## 2. Recommended interview order

If the interview is likely to be in English, the most effective order is:

1. project pack  
2. delivery and system basics  
3. backend fundamentals  
4. financial-domain understanding  
5. final role-fit summary

## 3. The overall positioning

You do not need to sound like a pure senior Java backend engineer with perfect theory coverage.

The stronger and more realistic positioning is:

`a fintech developer candidate with real asset-management system experience, practical delivery exposure, Java as the main backend line, and workable understanding of Python / React / deployment flow / system integration.`
