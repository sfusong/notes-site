# 资料口径说明

本资料包用于产品部竞聘学习，来源于今天培训会议转写稿、已校对稿、你已确认的术语，以及“产品部部分资料”目录下的流程文件、产品管理系统手册、销售文件模板、TA 要素表、风险评级、流动性风险评估、投资库风险管控材料、去纸化审批指引、理财登记系统数据元规范和产品说明书样例。

你的关键背景：你在科技部的工作本身就是对接产品部、参与筹建产品管理系统。因此竞聘叙事不应写成“从科技部零基础转产品部”，而应写成“长期参与产品部流程数字化建设，理解产品部从手工材料制作走向要素化、系统化过程中的真实痛点，现在希望进一步走到业务前台，把系统建设经验转化为产品全生命周期管理能力”。

## 1. 使用顺序

1. 先看 [06-一页纸速记.md](./06-一页纸速记.md)，建立整体框架。
2. 再看 [01-产品部全流程学习笔记.md](./01-产品部全流程学习笔记.md)，把流程完整学一遍。
3. 用 [02-产品部核心流程SOP.md](./02-产品部核心流程SOP.md) 对照记忆每个流程的触发条件、材料和系统动作。
4. 用 [03-高频术语卡片.md](./03-高频术语卡片.md) 纠正表达口径。
5. 用 [04-面试题库与回答模板.md](./04-面试题库与回答模板.md) 练习自然复述。
6. 用 [05-产品部流程优化建议.md](./05-产品部流程优化建议.md) 准备“产品管理系统筹建经验如何赋能产品部”。
7. 用 [08-深度校准与二次优化说明.md](./08-深度校准与二次优化说明.md) 了解这次二次优化新增了哪些口径。
8. 最后看 [07-待确认问题清单.md](./07-待确认问题清单.md)，决定是否向业务同事补问。

## 3. 关键校准结论

| 原先容易混淆的说法 | 建议使用的正式口径 | 说明 |
|---|---|---|
| 定投流程 | 投资经理初始设置流程/定投资经理流程 | 这里不是基金定投，而是设置产品主/副投资经理 |
| 五书 | 销售协议+四书 | 发行登记资料口径为“销售协议+四书” |
| 四书 | 产品说明书、投资协议书、风险揭示书、投资者权益须知 | 产品销售文件核心组成 |
| 三书 | 投资协议书、风险揭示书、投资者权益须知 | 申报材料中常与销售协议、说明书分开列示 |
| 去纸化 | 去纸化审批/OA 去纸化流程 | 用于内部审批、运营处理、归档留痕 |
| 风险评级表 | 理财产品风险评估及评级意见审批单/风险评级表 | 用于形成产品风险等级 |
| 流动性风险评估 | 主要针对开放式产品，封闭式产品不需要 | 资料流程表明确区分 |
| 投资库 | 常规投资库/主题型投资库 | 主题产品发行前需确认是否需新建主题投资库及风控指标 |
| 风控指标 | 投资库比例、计算规则、监控开始时间、超限处置等 | 不是发行后再补，而是申报/发行前要和风险合规、投决会衔接 |
| 理财登数据元 | 理财登记系统结构化字段 | 产品部工作不是只做 Word，而是管理监管报送字段 |
| 销售文件更新时间 | 系统用于销售文件更新、运营下载、报送和信披衔接的关键时间字段 | 具体上线状态需以当前系统为准 |
| 企业显然值 | 该份额的资产净值 | 结合 3309 母子份额说明书校准 |
| 复杂计费基准 | 扣除回购融资、产品运作及清算费用、增值税及附加税费后的资产净值 | 结合 3309 母子份额说明书校准 |

## 4. 阅读时要抓住的三条主线

| 主线 | 你要记住什么 | 面试价值 |
|---|---|---|
| 流程线 | 产品方案、产品委员会、定投资经理、申报登记、发行登记、渠道上架、存续期管理 | 证明你理解产品部全链路，不只会做系统 |
| 材料线 | 可行性评估报告、尽调报告、销售协议+四书、发行预告、风险评级表、流动性风险评估报告、导入模板 | 证明你知道产品部日常具体产出 |
| 系统线 | 产品管理系统、文档模板、产品要素导出表、TA 要素表、理财登报送、信披接口、OA 去纸化审批 | 证明你不是泛泛懂科技，而是参与过产品部流程数字化建设 |
| 风险线 | 投资库、主题投资库、风控指标、风险评级、流动性风险评估、投决会 | 证明你理解产品拓展不能只看卖点，还要看投资边界和风险承接 |

## 5. 使用边界

本资料包是竞聘学习版，不是制度原文，也不能替代正式流程制度。涉及接口上线时间、审批节点、经办人、系统字段名称、内部简称等易变化内容，应以产品部最新制度、系统页面和业务同事确认为准。
