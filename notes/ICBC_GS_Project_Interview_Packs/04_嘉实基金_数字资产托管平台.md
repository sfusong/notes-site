# 嘉实基金数字资产托管平台面试资料

## 1. 30 秒项目介绍

这是一个数字资产托管平台原型项目，采用前后端分离架构。  
我在里面做了两块：

- 用 `React` 实现用户、账户、转账、交易对手、资产管理等核心页面
- 用 `Node.js / Express + Sequelize + Swagger + JWT + Bcrypt` 实现后端服务、数据模型和接口鉴权

此外还接触了钱包托管、链上交互和智能合约相关 Web3 场景研究。

## 2. 可确认事实

来自 PPT 可以确认的内容：

- 前端：`HTML / CSS / JavaScript / React / Route / Component`
- 系统功能：用户系统、账户管理、转账管理、交易对手管理、资产管理
- 后端：`Node.js / Express`
- 接口：`RESTful API`
- ORM：`Sequelize`
- 文档：`Swagger`
- 鉴权：`JWT + Bcrypt`
- Web3：钱包托管、智能合约、热钱包

## 3. 技术和组件解释

### React

前端框架，主要用于：
- 页面组件化
- 路由切换
- 表单和列表展示
- 前后端联调

### Node.js

JavaScript 运行时，适合快速搭建后端服务。

### Express

Node.js 的 Web 框架，用于：
- 路由定义
- API 接口
- 请求处理

### RESTful API

一种接口设计风格，强调：
- 资源导向
- URL 清晰
- 前后端通过统一接口交互

### Sequelize

Node.js ORM，用来：
- 定义模型
- 映射数据库表
- 简化数据库 CRUD

### Swagger

接口文档工具，作用是：
- 让前后端对接口定义更清楚
- 方便测试
- 便于演示和联调

### JWT

JSON Web Token，用于登录后的身份令牌。

### Bcrypt

密码加密工具，一般用于：
- 密码哈希存储
- 避免明文密码

### 钱包托管 / 智能合约

这里你不要讲得太深，因为这份实习核心不是纯链开发。  
你只需要讲清楚：

- 你理解钱包托管场景
- 你接触过链上交互
- 你知道智能合约是链上业务逻辑承载方式

## 4. 架构理解

你可以用这版说法：

```text
React 前端
  -> 用户系统 / 账户管理 / 转账 / 资产页面
  -> 调用 Node.js/Express REST API

Express 后端
  -> Controller / 路由
  -> 业务逻辑
  -> Sequelize ORM
  -> 数据库

鉴权层
  -> Bcrypt 做密码哈希
  -> JWT 做登录后身份认证

Web3 研究方向
  -> 钱包托管
  -> 智能合约交互
```

## 5. 面试官最可能问的问题

### Q1. 这个平台本质上是什么？

回答：

本质上是一个数字资产托管平台原型，核心是验证用户、账户、转账、资产管理等基础能力在前后端分离架构下如何实现，同时探索钱包托管和链上交互场景。

### Q2. 你在里面做了什么？

回答：

我主要做了前后端原型开发。前端用 React 实现业务页面，后端用 Node.js/Express 提供 REST API，并结合 Sequelize、JWT、Bcrypt 和 Swagger 完成数据模型、鉴权和接口文档能力。

### Q3. 为什么用 JWT 和 Bcrypt？

回答：

因为这是比较标准的用户认证方案。Bcrypt 用来安全存储密码，JWT 用来在登录后维持会话和保护接口访问。

### Q4. Swagger 在这里起什么作用？

回答：

它帮助统一前后端接口理解，也方便测试和演示。对原型项目来说，这一点其实很重要，因为可以让接口定义更清晰。

### Q5. 你说接触了 Web3，这部分具体是什么？

建议回答：

这部分我不会夸大成自己主导了完整链上系统，更准确的说法是，我在原型开发之外，也参与了钱包托管、链上交互和智能合约应用场景的研究，用于支持平台未来方向判断。

### Q6. 前后端分离给这个项目带来什么好处？

回答：

前后端分离有利于功能模块清晰划分、并行开发和后续扩展。尤其在原型阶段，这样可以更快验证产品能力。

## 6. 如果追问到更细的技术点

### 1. Sequelize 是怎么用的？

稳妥答法：

主要用于定义数据模型以及处理和数据库的交互，帮助降低直接写 SQL 的工作量。

### 2. 你们是不是接了真实区块链？

安全答法：

这段经历更适合讲成“围绕钱包托管和智能合约交互做了场景研究与原型探索”，不要硬说成大规模真实链上生产系统。

### 3. 这个项目最有价值的地方是什么？

回答：

一方面是把前后端分离的托管平台原型完整搭起来，另一方面是让我第一次系统接触鉴权、ORM、接口文档和 Web3 托管场景的结合。
