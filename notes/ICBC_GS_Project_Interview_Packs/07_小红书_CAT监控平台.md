## 3. CAT 是什么

你要能用一句话解释：

`CAT 是一个实时应用监控与告警平台，用来收集应用监控数据、做统计聚合、展示报表并触发告警。`

## 4. 组件和技术解释

### Maven 多模块

Maven 是 Java 构建工具，多模块意思是系统按功能拆成多个子模块，例如：
- 客户端上报
- 服务端聚合
- 告警
- 管理后台

### 监控消息采集

指应用把运行过程中产生的监控事件、事务、心跳等数据上报到平台。

### 报表聚合

指服务端把大量原始监控消息按时间、维度和模型做汇总，生成可展示的统计结果。

### 后台展示

指管理端或报表端把聚合后的结果做成页面，供运维或开发查看。

### 告警链路

指系统在满足某些阈值或异常条件时，触发告警通知。

## 5. 监控模型怎么讲

如果问到 CAT 的监控模型，你可以答得简单一点：

- `Transaction`：一次业务调用或流程
- `Event`：某个关键事件
- `Heartbeat`：系统存活与健康信息
- `Metric`：指标类数据

如果你不确定 `Metric` 细节，就别展开太深。

## 6. 面试可用架构理解

基于 README，你可以这样讲：

```text
应用客户端
  -> 上报监控消息

服务端核心
  -> 接收消息

  -> 生成报表

管理后台

cat-client -> cat-consumer -> cat-home / cat-alarm
```

## 7. 面试官最可能问的问题

### Q1. 你在这个项目里到底做了什么？

回答：

我参与的是基于开源 CAT 的监控平台升级适配工作，在 Java/Maven 多模块代码库里做后端模块相关开发和调试，接触了监控消息处理、报表聚合和告警链路。

### Q2. 你理解的监控平台核心价值是什么？

回答：

核心价值是尽快发现问题、缩短定位时间，并帮助开发和运维理解系统运行状态。对业务系统来说，尤其是复杂系统，监控和告警是稳定性保障的基础能力。

### Q3. 监控平台的核心链路是什么？

回答：

采集、传输、聚合、展示、告警。也就是先收集运行数据，再做处理和汇总，最后展示出来并在异常时发告警。

### Q4. 为什么监控平台需要多模块？

回答：

因为采集、聚合、展示、告警本身就是不同职责，拆成多模块能让系统结构更清晰，也便于分别演进。

### Q5. 你说接触报表聚合，报表聚合为什么难？

回答：

因为原始监控消息量很大，报表聚合要在时间粒度、统计口径和性能之间找平衡，同时还要保证展示结果对用户有意义。

### Q6. 这个项目对你最大的收获是什么？

回答：

它让我第一次比较系统地理解可观测性平台的完整链路，不只是写一个单独业务模块，而是看到监控从消息采集到展示和告警的整体结构。
