使用建议：

- 先背每份里的“30 秒项目介绍”
- 再看“组件/技术解释”
- 再重点准备“高频追问与回答”
- 最后只挑你最有把握的架构细节来讲

如果是英文面试，建议顺序可以改成：

- 先看对应英文版里的 `30-second version`
- 再看 `1-minute version`
- 再准备 `Architecture / data flow / trade-off`
- 最后只保留你能稳定说清楚的技术边界，不要为了“显得懂”去过度延展

说明：

- 我把内容分成了“可确认事实”和“面试可用的合理推断”两层。
- 如果是我根据现有材料和常见工程实现方式补出来的，会明确告诉你这是“合理推断”，你自己决定是否采用。
- 不建议把所有细节都背成标准答案，最好理解后再转成你自己的话。
