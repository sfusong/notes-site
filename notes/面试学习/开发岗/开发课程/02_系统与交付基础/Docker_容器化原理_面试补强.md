# Docker 容器化原理面试补强

这份只补一个洞：

`如果高盛工银 onsite 继续问 Docker / 容器化，你至少能讲出原理、价值和几个常用命令。`

---

## 一、最短答案

如果面试官问：

`Docker 的理念或者原理是什么？`

你可以先答：

`Docker 是一种容器化技术，它把应用程序和运行依赖一起打包成镜像，再通过容器运行起来。容器不像虚拟机那样完整虚拟一套操作系统，而是共享宿主机内核，通过 namespace 做隔离，通过 cgroups 做资源限制，所以更轻量、启动更快，也更适合标准化部署和测试环境。`

这段先背熟，够你接住第一问。

---

## 二、Docker 到底解决什么问题

Docker 最核心不是“让代码变快”，而是解决：

- 本地、测试、生产环境不一致
- 依赖安装和版本冲突
- 部署过程不标准
- 应用迁移到新环境成本高

一句话：

`Docker 的价值是把应用运行环境标准化。`

---

## 三、镜像和容器

## 1. 镜像 image

`镜像是应用和依赖的静态打包结果。`

比如：

- Java 运行环境
- 应用 jar 包
- 启动命令
- 配置文件或默认环境

都可以被组织进镜像。

## 2. 容器 container

`容器是镜像运行起来之后的实例。`

可以这样类比：

- 镜像像 class
- 容器像 object

同一个镜像可以启动多个容器。

---

## 四、容器和虚拟机的区别

## 1. 虚拟机

虚拟机通常包括：

- 虚拟硬件
- 完整 Guest OS
- 应用和依赖

所以它更重，启动更慢。

## 2. 容器

容器通常包括：

- 应用
- 应用依赖
- 隔离的进程、网络、文件系统视图

但它共享宿主机内核。

所以它更轻，启动更快。

## 3. 面试稳妥说法

`我理解 Docker 更偏操作系统层面的轻量虚拟化，而不是传统虚拟机那种硬件层虚拟化。容器共享宿主机内核，所以它通常比虚拟机轻量，但隔离程度也不是完全等同于一台独立虚拟机。`

---

## 五、namespace 和 cgroups

这两个词不需要讲太深，但一定要知道作用。

## 1. namespace

`namespace 负责隔离。`

常见隔离包括：

- 进程隔离
- 网络隔离
- 文件系统视图隔离
- 主机名隔离

你可以说：

`namespace 让容器里的进程感觉自己在一个独立环境里运行。`

## 2. cgroups

`cgroups 负责资源限制和统计。`

比如限制：

- CPU
- 内存
- IO

你可以说：

`cgroups 用来限制一个容器能使用多少宿主机资源，避免某个容器把整台机器资源吃满。`

## 六、Docker 的 IO 怎么理解

如果面试官提到 `IO`，不要慌，先抓住三层。

## 1. 文件系统

Docker 镜像通常是分层的。

容器运行后，会在镜像层上增加一个可写层。

所以：

- 镜像层通常只读
- 容器可写层保存运行时变更
- 容器删除后，可写层内容通常也会丢

## 2. 持久化

真正重要的数据，一般不只放在容器可写层。

常见做法：

- volume
- bind mount
- 外部数据库或对象存储

## 3. IO 本质

容器不是凭空拥有独立硬件。

文件 IO、网络 IO 最终还是落到宿主机资源和内核能力上。

面试稳妥说法：

`我理解 Docker 的 IO 不是完全独立的一套硬件 IO。容器里的文件系统有镜像层和可写层，真正需要持久化的数据一般会通过 volume 或挂载目录来处理，底层文件和网络 IO 仍然依赖宿主机内核和资源。`

---

## 七、Docker 网络怎么理解

最基本的三句话：

- 容器有自己的网络命名空间
- 容器里服务监听端口
- 通过端口映射或容器网络与外部通信

比如：

```bash
docker run -p 8080:8080 app-image
```

含义是：

`把宿主机的 8080 端口映射到容器里的 8080 端口。`

面试稳妥说法：

`Docker 会给容器提供隔离的网络环境，应用在容器内监听端口，再通过端口映射或容器网络和外部系统、其他容器通信。`

## 八、常用命令够用版

## 1. 看镜像

```bash
docker images
```

## 2. 拉镜像

```bash
docker pull nginx
```

## 3. 启动容器

```bash
docker run -d --name my-nginx -p 8080:80 nginx
```

含义：

- `-d`：后台运行
- `--name`：给容器命名
- `-p`：端口映射

## 4. 看容器

```bash
docker ps
docker ps -a
```

## 5. 看日志

```bash
docker logs my-nginx
```

## 6. 进入容器

```bash
docker exec -it my-nginx bash
```

## 7. 停止和删除

```bash
docker stop my-nginx
docker rm my-nginx
```

## 8. 构建镜像

```bash
docker build -t my-app:1.0 .
```

## 九、怎么和你的项目挂钩

你不用说宁银的项目大量使用 Docker，除非确实用过。

更稳的说法是：

`我之前在项目里更多接触的是传统企业内网环境、Jenkins Pipeline、Linux 脚本、制品和部署流程。Docker 我不是在生产里深度负责过，但我理解它解决的是应用环境标准化和部署一致性问题。对自动化测试或新系统交付来说，如果后续把测试依赖、Mock 服务或应用服务容器化，可以减少环境搭建成本，也更利于 CI/CD 流水线。`

这句话诚实，但不露怯。

## 十一、最后背这 6 句话

1. Docker 是容器化技术，用来打包应用和依赖，保证环境一致。
2. 镜像是静态模板，容器是镜像运行起来后的实例。
3. 容器共享宿主机内核，不是完整虚拟一套操作系统。
4. namespace 负责隔离，cgroups 负责资源限制。
5. 容器 IO 依赖镜像层、可写层和宿主机资源，持久化数据一般用 volume。
6. Docker 对 CI/CD 的价值在于环境标准化、部署一致性和测试环境快速拉起。
