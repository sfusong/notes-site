# Tomcat、Maven、打包、JDK/JRE/JVM

这份笔记回答 4 个问题：

1. `Tomcat` 到底是什么
2. `Maven` 是什么
3. `Maven` 和“打包”是什么关系
4. `JDK / JRE / JVM` 到底有什么区别

目标是让你在开发岗面试里，不只是背定义，而是真能把这些东西串起来讲。

---

## 1. Tomcat 到底是什么

## 1.1 最简单的理解

`Tomcat` 是一个：

- Java Web 应用服务器 / Servlet 容器

如果这个定义太抽象，你可以先这样理解：

`Tomcat 是一个专门用来运行 Java Web 应用的服务。`

它能做的事情包括：

- 接收 HTTP 请求
- 把请求交给你的 Java Web 应用处理
- 再把响应返回给客户端

所以你可以把它想成：

`“托管 Java Web 应用的运行环境”`

---

## 1.2 为什么需要 Tomcat

假设你写了一个 Java Web 应用，
这个应用要对外提供接口、接收浏览器请求、返回页面或 JSON。

那它不能只是一堆 Java 代码放在那里，
它需要一个“能接 HTTP 请求并调用你程序”的运行环境。

Tomcat 就是干这个的。

所以你可以理解为：

- 你的代码负责业务逻辑
- Tomcat 负责把 Web 请求接进来，并把你的应用跑起来

---

## 1.3 Tomcat 和 war 的关系

这个地方和你前面问的 `war` 连起来了。

传统 Java Web 应用常见方式是：

- 把项目打成 `war`
- 部署到 `Tomcat`
- 由 `Tomcat` 启动并运行这个应用

所以你可以记：

- `war` 是“包”
- `Tomcat` 是“运行这个包的容器/环境”

---

## 1.4 为什么现在很多 Spring Boot 项目不太强调 Tomcat 了

因为现在很多 Spring Boot 项目会：

- 把内嵌 Tomcat 一起打进 `jar`

这样你只需要：

```bash
java -jar app.jar
```

应用就能直接跑起来，
不需要再单独安装一个外部 Tomcat 去部署 `war`。

所以现在常见两种模式：

### 传统模式

- 项目打成 `war`
- 部署到外部 Tomcat

### Spring Boot 常见模式

- 项目打成 `jar`
- `jar` 里内嵌 Tomcat
- 直接 `java -jar`

---

## 1.5 面试时怎么说

如果问：

“Tomcat 是什么？”

你可以答：

“Tomcat 可以理解为运行 Java Web 应用的服务器或容器。它负责接收 HTTP 请求，并把请求交给应用处理。传统 Java Web 项目经常会把应用打成 war 包再部署到 Tomcat 上运行；而现在很多 Spring Boot 项目会把内嵌 Tomcat 直接打进 jar，所以部署方式更简单。”

---

## 2. Maven 是什么

## 2.1 最简单的理解

`Maven` 是 Java 生态里非常常见的：

- 项目构建工具
- 依赖管理工具

你先记住一句：

`Maven 帮你管理项目怎么编译、怎么打包、用哪些依赖。`

---

## 2.2 Maven 主要干什么

Maven 常见做的事情有：

- 管理依赖
- 编译代码
- 跑测试
- 打包生成 jar/war
- 执行构建流程

所以它不是单纯“下载依赖”的工具，
它更像是：

`“Java 项目的构建总管”`

---

## 2.3 依赖管理是什么意思

以前如果没有 Maven，
你可能要自己手动下载各种 jar 包，
放进项目里。

有了 Maven 之后，
你只需要在 `pom.xml` 里写清楚你依赖什么，
Maven 会帮你下载和管理这些库。

比如：

- Spring Boot
- MyBatis
- Jackson
- 数据库驱动

这些都可以通过 Maven 管理。

---

## 2.4 构建工具是什么意思

Maven 不只是“下载包”，它还负责：

- 怎么编译源码
- 怎么组织输出目录
- 怎么跑测试
- 怎么打包

你可以理解为：

`“它规定了一个 Java 项目从源码变成可运行产物的流程。”`

---

## 2.5 面试时怎么说

如果问：

“Maven 是什么？”

你可以答：

“Maven 是 Java 项目里常见的构建和依赖管理工具。一方面它通过 pom.xml 管理项目依赖，另一方面也负责项目的编译、测试和打包流程，比如最终生成 jar 或 war。对我来说，Maven 不只是下载依赖，而是整个 Java 项目构建过程的重要工具。”

---

## 3. Maven 和“打包”是什么关系

## 3.1 什么叫打包

打包最简单的理解就是：

- 把源码、编译结果、资源文件、必要配置等
- 按照一定格式整理成一个可运行或可部署的产物

常见结果就是：

- `jar`
- `war`

---

## 3.2 Maven 怎么参与打包

Maven 会根据项目配置和插件，
执行构建流程，
最后把项目打成：

- `jar`
- 或 `war`

比如最常见的命令：

```bash
mvn package
```

这个命令通常意味着：

- 编译代码
- 执行测试（视配置而定）
- 最终生成打包产物

所以你可以理解成：

- `打包` 是一个动作
- `Maven` 是执行这个动作的重要工具

---

## 3.3 结合你们公司的流程怎么理解

你说在“鲁班”里可以配置：

- Maven 版本
- JDK 版本

这就说明：

- 你们公司的发布流程里，构建步骤和 Java 环境是明确受控的
- Maven 负责构建/打包逻辑
- JDK 负责提供 Java 编译和运行环境

也就是说，打包不是“凭空产生”的，
而是：

- 在指定的 JDK 环境下
- 用指定版本的 Maven
- 按既定流程完成构建和打包

---

## 3.4 面试时怎么说

如果问：

“Maven 和打包是什么关系？”

你可以答：

“打包是把项目整理成可部署产物的过程，比如生成 jar 或 war。Maven 是执行这个过程的重要工具，它不仅管理依赖，也负责项目的编译、测试和打包。在实际项目里，我们会通过 Maven 把源码构建成最终可部署的制品。”

---

## 4. JDK / JRE / JVM 的区别

这是开发岗很喜欢问的经典题。

先给你最简单的记法：

- `JVM`：让 Java 程序跑起来的虚拟机
- `JRE`：运行 Java 程序需要的环境
- `JDK`：开发 Java 程序需要的工具包

如果你把这三层关系记住，已经能答大部分面试。

---

## 4.1 JVM 是什么

`JVM` 全称：

- `Java Virtual Machine`

它的核心作用是：

- 执行 Java 字节码

你可以把它理解成：

`“Java 程序真正运行的地方”`

Java 源码不是直接跑的，
而是先编译成字节码，
再由 JVM 去执行。

所以 JVM 是运行层。

---

## 4.2 JRE 是什么

`JRE` 全称：

- `Java Runtime Environment`

它的作用是：

- 给 Java 程序提供运行环境

JRE 里面会包含：

- JVM
- 运行 Java 程序需要的基础类库

所以你可以理解成：

- `JRE = JVM + 运行所需基础环境`

它适合：

- 运行 Java 程序

但不一定适合开发 Java 程序。

---

## 4.3 JDK 是什么

`JDK` 全称：

- `Java Development Kit`

它的作用是：

- 给开发者提供完整的 Java 开发工具

JDK 里面通常包含：

- JRE
- 编译器 `javac`
- 调试、打包等开发工具

所以你可以理解成：

- `JDK = JRE + 开发工具`

它适合：

- 写 Java
- 编译 Java
- 打包 Java

---

## 4.4 三者关系怎么记

最推荐你记成这个层级：

`JDK > JRE > JVM`

意思是：

- `JDK` 里包含 `JRE`
- `JRE` 里包含 `JVM`

所以：

- 开发需要 `JDK`
- 单纯运行只需要 `JRE`
- 真正执行字节码的是 `JVM`

---

## 4.5 一个生活化类比

你可以这样记：

- `JVM` 像发动机
- `JRE` 像整辆能开的车
- `JDK` 像修车和造车的完整工具箱

如果你只是坐车跑起来：

- 需要 `JRE`

如果你自己要造车、修车：

- 需要 `JDK`

---

## 4.6 为什么你们投产要配置 JDK 版本

因为：

- Java 项目的编译和运行都依赖 Java 环境
- 不同 JDK 版本可能影响兼容性

在你们公司的发布平台上指定 JDK 版本，
本质上是在控制：

- 构建环境一致性
- 运行环境一致性

这也是标准工程实践的一部分。

---

## 4.7 面试时怎么说

如果问：

“JDK、JRE、JVM 有什么区别？”

你可以答：

“JVM 是 Java 虚拟机，负责真正执行 Java 字节码；JRE 是 Java 运行环境，里面包含 JVM 和运行程序需要的基础类库；JDK 则是在 JRE 基础上再加上编译、调试等开发工具，所以开发 Java 程序通常需要 JDK。简单理解就是 JDK 包含 JRE，JRE 包含 JVM。”

这就是很标准的答案。

---

## 5. 把这些概念串起来

你现在可以把这几个概念串成一条链：

1. 你写 Java 代码
2. 用 `JDK` 和 `Maven` 做编译和打包
3. 生成 `jar` 或 `war`
4. 如果是传统 Web 项目，可能部署到 `Tomcat`
5. 如果是 Spring Boot，也可能直接 `java -jar`
6. 最终程序运行在 `JRE/JVM` 之上

这就是一条完整的理解路径。

---

## 6. 面试里的组合回答

如果面试官把这些概念连着问，你可以这样答：

“Tomcat 可以理解为运行 Java Web 应用的服务器或容器，传统 Java Web 项目常把应用打成 war 部署到 Tomcat 上。Maven 是 Java 项目里常见的构建和依赖管理工具，负责依赖管理、编译和打包，最终可以生成 jar 或 war。至于 Java 运行环境，JVM 负责执行字节码，JRE 提供运行 Java 程序所需环境，而 JDK 在 JRE 基础上再提供编译和调试等开发工具，所以开发时通常用 JDK。”

---

## 7. 你现在最应该记住的 5 句话

1. `Tomcat` 是运行 Java Web 应用的服务器/容器
2. `Maven` 是 Java 项目的构建和依赖管理工具
3. `打包` 就是生成 jar 或 war 这类可部署产物
4. `war` 常部署到 Tomcat，`jar` 常可直接运行
5. `JDK > JRE > JVM`
