# React：开发岗面试够用版

这份笔记的目标不是把你训练成前端专家，而是让你在金融开发岗面试里：

- 能讲清 React 是什么
- 能把你做过的 React 项目讲顺
- 能回答几类最常见的 React 基础问题
- 不在“React 是加分项”这块失分

## 一、先记住你的定位

对你来说，React 在开发岗里的定位是：

- `加分项`
- `项目支撑项`
- `前后端协作理解项`

它不是你这条线的主战场，主战场依然是：

- Java
- 系统交付
- 金融系统项目

所以你的目标不是把 React 补到很深，而是做到：

`能讲项目、能答基础、能说明白自己是怎么和后端系统一起把页面做起来的。`

## 二、React 最简单是什么

最简单地说：

`React 是一个用来构建前端用户界面的 JavaScript 库。`

它最大的特点是：

- 把页面拆成组件
- 用状态驱动界面更新
- 更适合做交互复杂的单页应用

如果面试官问你“React 是什么”，最稳的答法是：

“我理解 React 是一个基于组件化思想构建前端界面的库。它会把页面拆成可复用组件，再通过状态变化驱动界面更新，所以比较适合交互较多、前后端分离的业务系统。”

## 三、你至少要懂的 5 个概念

### 1. 组件 Component

React 页面不是一整块写完，而是拆成很多组件。

比如一个系统页面可能拆成：

- 搜索栏组件
- 表格组件
- 弹窗组件
- 表单组件
- 分页组件

组件化的好处是：

- 复用
- 维护方便
- 页面结构更清晰

### 2. Props

`props` 可以理解成：

`父组件传给子组件的参数。`

比如：

- 表格组件需要列定义
- 弹窗组件需要标题和开关状态

这些都可以通过 `props` 传进去。

### 3. State

`state` 可以理解成：

`组件自己管理的数据状态。`

比如：

- 当前输入框内容
- 当前弹窗是否打开
- 当前表格数据
- 当前加载状态

React 的界面更新，本质上就是：

`state 变了，页面跟着重新渲染。`

### 4. 事件处理

比如：

- 点击按钮
- 输入内容
- 提交表单

React 会用事件处理函数去接这些交互，再触发状态更新或接口调用。

### 5. 生命周期 / Hook

你不用讲特别深，但至少要知道：

- 现在函数组件里常用 `useState`
- 请求接口、初始化数据常用 `useEffect`

如果面试官问到 React 基础，这是非常常见的一层。

## 四、React 在项目里通常怎么工作

最典型的模式是：

1. 用户进入页面
2. React 组件渲染页面结构
3. 页面初始化时调用后端接口获取数据
4. 用户搜索、点击、编辑
5. 前端把参数发给后端
6. 后端返回结果
7. React 根据新数据更新页面

所以 React 在项目里的价值，不只是“把页面画出来”，而是：

- 承接交互
- 管理页面状态
- 和后端 API 协作

## 五、你自己的 React 项目怎么讲

你最适合讲 React 的几个项目有：

### 1. 产品生命周期管理系统

可以这样讲：

“这个系统前端用的是 React，我参与的是围绕业务规则把页面交互和后端能力串起来，比如关键节点提醒、事件管理和多角色协作场景。对我来说 React 这部分重点不是做视觉效果，而是把表单、列表、状态切换和后端接口联动起来，让业务能真正用起来。”

### 2. 嘉实基金数字资产托管平台原型

可以这样讲：

“那段经历里我更完整地做过 React 页面开发，包括用户、账户、转账、交易对手和资产管理这些页面。它让我比较熟悉组件化开发、页面状态管理以及和后端接口联调的基本方式。”

## 六、面试最常见的 React 问题

### 1. React 和传统 jQuery / 直接操作 DOM 的区别

最稳答法：

“我理解 React 更强调组件化和状态驱动界面更新，而不是手动频繁操作 DOM。这样页面结构会更清晰，复杂交互和前后端分离场景也更容易维护。”

### 2. 什么是组件化

最稳答法：

“就是把页面拆成可复用、职责清晰的小模块，比如表格、表单、弹窗、分页等，每个组件只关注自己的输入和输出，方便复用和维护。”

### 3. props 和 state 的区别

最稳答法：

“props 更像组件外部传进来的参数，state 更像组件内部维护的状态。props 通常由父组件控制，state 则会随着用户交互和接口结果变化而更新页面。”

### 4. React 怎么和后端联动

最稳答法：

“通常是在组件初始化或用户触发操作时发起接口请求，把查询条件或表单数据传给后端，再根据返回结果更新页面状态，比如刷新列表、更新详情或提示提交结果。”

### 5. 你在 React 里最常做的是什么

你可以答：

“对我来说，更多是围绕业务页面做表单、列表、详情和弹窗这类常见交互，并和后端接口联调，保证页面状态和业务流程能跑通。”

## 七、你不需要硬讲太深的内容

如果你没准备好，不建议主动展开：

- React Fiber
- 虚拟 DOM 底层细节
- 复杂状态管理库源码
- 性能优化很深的话题
- 前端工程化特别细的构建链

因为对你这条开发岗路线来说，React 只是加分项，不是主考场。

## 八、最适合你的定位

你最稳的 React 定位不是：

- “我是资深前端”

而是：

`我具备 React 页面开发和前后端联调能力，能在业务系统建设中承担前端交互实现和接口联动工作。`

## 九、最后记住 5 句话

- React 是一个基于组件化思想构建前端界面的库
- 它的核心是组件、props、state 和状态驱动更新
- 它很适合做前后端分离的业务系统页面
- 对你来说，重点是讲清项目里的交互实现和接口联调
- 在开发岗里，React 是加分项，不是主战场
