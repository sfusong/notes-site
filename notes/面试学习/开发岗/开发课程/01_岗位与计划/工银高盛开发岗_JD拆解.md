# 工银高盛开发岗 JD 拆解

对应原始岗位：
工银高盛理财.txt（外部文件：工银高盛理财.txt，站内未导入）

## 一、先讲一句话判断

这份 JD 对你的吸引点很明确：

- `金融背景匹配`
- `Java 是主线，但 Python / React 是加分项`
- `既看开发，也看系统交付和协作`

它不是只招“只会写接口的人”，而是更偏：

`能在金融机构里做关键业务系统建设、联调、发布和持续迭代的软件开发工程师。`

## 二、JD 在要什么

### 1. 核心编程能力

重点是：

- `Java`
- 数据结构和算法
- Python 或 React 加分

这说明它希望候选人主语言足够稳，同时能接受一定的多栈配合。

### 2. 系统运行与中间件认知

明确点名了：

- Linux
- Windows Server
- Nginx
- Tomcat
- Redis
- MySQL

这说明它希望你不是只会写业务代码，也要知道服务大概怎么运行、怎么部署、怎么接入。

### 3. 工程流程能力

明确点名了：

- SDLC
- CI/CD
- IaC
- Git

这说明它很重视：

- 团队协作
- 代码版本管理
- 构建发布流程
- 基础交付认知

### 4. 金融业务与沟通能力

优先条件里提了：

- 资管 / 金融服务经验
- 金融市场 / 金融资产 / 风险管理经验
- 和业务、技术团队沟通
- 英语口语和书面表达

也就是说，这不是纯外包开发岗，而是希望你能真正进入金融业务语境。

## 三、你当前的匹配点

### 1. 强匹配

- `资管 / 理财子背景`
- `TA、产品管理系统、批处理、接口切换、投产验证`
- `Java Spring Boot`
- `Python`
- `React`
- `Linux`
- `Jenkins / CI-CD 基础认知`
- `跨部门、供应商、业务、技术协同`

这部分其实已经构成了你最有竞争力的主线：

`有金融机构真实系统建设经验的技术型候选人`

### 2. 中匹配

- `Tomcat / Nginx / Redis`
- `Git`
- `SDLC`
- `数据库与系统交付认知`

这部分你可以讲，但需要继续补到更稳。

### 3. 当前短板

最明显的短板有 6 个：

1. `3 年以上纯开发经验叙事还不够硬`
2. `Java 基础/集合/并发/JVM 等底层题还没系统补`
3. `数据结构与算法是显性要求`
4. `Windows Server` 几乎还没准备
5. `IaC（Ansible / Terraform）` 目前只是 JD 知道，没形成答题能力
6. `英语口语` 是潜在风险项

## 四、这份 JD 对你意味着什么

你的打法不应该是：

- 把自己包装成“纯 Java 老后端”

而应该是：

`把自己讲成有资管业务背景、能做系统建设与交付、主语言是 Java、并具备一定 Python/React 和发布流程认知的金融开发候选人。`

这个定位更真实，也更适合你当前履历。

## 五、这条线最值得优先补的能力

按优先级，我建议是：

1. `Java / Web 运行基础`
2. `CI/CD / Git / 发布链路`
3. `Nginx / Tomcat / Redis / MySQL`
4. `TA 自动化测试项目讲法`
5. `数据结构与算法基础`
6. `Linux / Windows Server / IaC 认知`

## 六、面试里最可能考你的几类题

### 1. 基础技术题

- jar 和 war 区别
- Spring Boot 为什么常打 jar
- Tomcat 和 Nginx 区别
- Maven 是什么
- JDK / JRE / JVM 区别

### 2. 交付流程题

- 你理解的 CI/CD 是什么
- 你们公司怎么发布
- Jenkins Pipeline 是什么
- 制品、制品库、回滚分别是什么

### 3. 项目题

- 产品生命周期管理系统你做了什么
- TA 自动化测试到底怎么实现
- TA 批前预警稽核平台解决什么问题
- 你在供应商系统里承担的真实价值是什么

### 4. 软性与场景题

- 怎么和业务沟通
- 怎么和测试 / 供应商协作
- 如何面对生产问题
- 如何看待版本上线与回滚

## 七、你后面学习时最该记住的一句话

不是去硬凹成“我就是很强的纯后端”，而是：

`把自己训练成一个能讲清 Java 基础、系统交付链路、中间件基本认知和金融项目经验的开发岗候选人。`
