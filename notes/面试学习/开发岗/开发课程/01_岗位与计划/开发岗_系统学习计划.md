## 一、这份计划的目标

目标不是“全栈全会”，而是把你训练成：

`在金融开发岗面试里，能够用真实项目撑住简历，并在系统基础、交付链路和常见中间件问题上讲得稳的候选人。`

## 三、你当前的学习策略建议

### 不要做的

- 不要一上来全面铺开学 Java 全家桶
- 不要把时间全花在很深的源码原理上
- 不要试图短时间把自己包装成 5 年纯后端

### 应该做的

- 先把 JD 明写的系统与交付基础补稳
- 再把你的金融项目讲顺
- 再把项目绑定的后端技术骨架讲顺
- 最后再补 Java / 算法 / 中间件这些显性短板

## 补充说明：React 在这条线里的定位

React 在工银高盛这类 JD 里属于：

- `明确加分项`
- 不是主语言主线

所以你的目标不是把 React 补成前端专家，而是做到：

- 能讲清楚你在 React 项目里做了什么
- 能答基础概念题
- 能把前后端协作和组件化思路讲顺

## 五、你当前最值得记住的一句话

这条线最好的胜法不是“装成纯八股后端”，而是：

`把自己训练成一个具备资管业务背景、能讲清系统交付链路、能讲清核心项目、并在 Java 与中间件基础题上不露怯的金融开发候选人。`
