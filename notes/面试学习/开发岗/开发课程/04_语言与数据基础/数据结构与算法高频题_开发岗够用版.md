# 数据结构与算法高频题：开发岗够用版

这份笔记的目标不是把你训练成刷题型选手，而是让你在开发岗面试里：

- 知道常见数据结构分别是什么
- 知道它们分别适合解决什么问题
- 能把时间复杂度、空间复杂度讲出最基础的层次
- 能对二分查找、排序、遍历这类高频算法题有答题框架

## 一、先讲一句话定位

对你现在的目标岗位来说，数据结构与算法的要求更像：

`不能完全不会，但也不是一定要刷到大厂算法面深度。`

所以你的目标不是：

- 每题都手写最优解

而是：

- 先把常见结构和经典思路讲明白
- 碰到基础题时，知道从哪里下手

## 二、先记住两个最基础的概念

## 1. 时间复杂度

最简单理解：

`算法执行所花的时间会随着数据规模怎么增长。`

你现在先记最常见的几个：

- `O(1)`：基本不随数据规模增长
- `O(log n)`：增长比较慢
- `O(n)`：线性增长
- `O(n log n)`：常见于较高效排序
- `O(n^2)`：双重循环常见

### 怎么用很口语的话讲

比如：

- 数组按下标取值通常接近 `O(1)`
- 把一整个数组看一遍是 `O(n)`
- 两层嵌套循环经常是 `O(n^2)`

面试最稳答法：

“时间复杂度描述的是算法运行时间随数据规模增长的趋势。实际面试里我一般会先判断有没有循环、嵌套循环、递归或二分这类特征，再去估一个大致复杂度，比如 O(n)、O(n log n) 或 O(n^2)。”

## 2. 空间复杂度

最简单理解：

`算法为了完成任务，额外占用了多少空间。`

比如：

- 原地交换元素，额外空间很少
- 额外开一个新数组、哈希表、递归栈，就会增加空间复杂度

面试最稳答法：

“空间复杂度主要看算法除了输入本身之外，还额外使用了多少存储空间，比如是否额外开数组、集合或递归栈。”

## 三、最常见的 6 类数据结构

## 1. 数组 Array

最简单理解：

`一段连续的数据结构，按下标访问很方便。`

特点：

- 查某个位置快
- 中间插入、删除通常不方便

最适合的问题：

- 按索引访问
- 顺序遍历
- 固定或相对稳定结构的数据

## 2. 链表 Linked List

最简单理解：

`每个节点连着下一个节点。`

特点：

- 插入删除节点更灵活
- 按位置访问不如数组直接

最适合的问题：

- 频繁插入、删除
- 节点重组
- 双指针类题目

## 3. 栈 Stack

最简单理解：

`后进先出。`

就像一摞盘子，最后放上去的先拿下来。

最适合的问题：

- 括号匹配
- 表达式处理
- 回退、撤销
- 递归调用的理解

## 4. 队列 Queue

最简单理解：

`先进先出。`

就像排队。

最适合的问题：

- 按顺序处理任务
- 广度优先搜索
- 消息消费

## 5. 哈希表 Hash Table / HashMap

最简单理解：

`通过 key 快速查 value 的结构。`

最适合的问题：

- 快速查找
- 去重
- 统计频次
- 建映射关系

这类题在算法里非常高频，因为很多“看起来要双重循环”的题，往往能靠哈希表优化到更低复杂度。

## 6. 树 Tree

你现在先建立最基础认知就够：

- 树是层级结构
- 二叉树是最常见一类

最适合的问题：

- 层级关系
- 遍历
- 搜索

## 四、你最该先会的 5 类算法思路

## 1. 遍历

这是最基础的算法思维。

最常见就是：

- 把数组看一遍
- 把链表走一遍
- 把树遍历一遍

很多题的第一步根本不是什么高级技巧，而是：

`先遍历，把信息拿全。`

## 2. 双指针

双指针非常常见，尤其是数组、链表问题。

最简单理解：

- 两个指针一起走
- 速度相同或不同
- 分别从两端或一前一后出发

最适合的问题：

- 有序数组查找
- 去重
- 快慢指针找链表中点、判断环

## 3. 哈希优化

很多题最直接的暴力解法是：

- 两层循环

但如果你想提升效率，常见思路就是：

- 先用哈希表存信息
- 再用一次遍历快速查

这类思路非常高频，也很适合开发岗面试。

## 4. 二分查找

二分查找的关键不是“会背模板”，而是：

`你有没有识别出“这个问题有序，可以砍半找”。`

最适合的问题：

- 有序数组查找某个值
- 找边界值
- 某些单调性问题

### 最基础答法

“二分查找的前提通常是数据有序或问题具备单调性。核心思路是每次取中间位置，比目标值大就往左，小就往右，把搜索范围不断缩小。”

## 5. 排序

你现在不用背所有排序源码，但至少要知道：

- 冒泡、选择、插入：基础排序
- 快排、归并、堆排：更经典
- Java 实际开发里通常不会手写排序，而是调用库

但面试官问你“常见排序有哪些”，你不能完全空。

最稳答法：

“基础排序里常见冒泡、选择、插入，复杂度通常较高；更高效的经典排序有快速排序、归并排序等。实际业务开发里更多是理解排序思路和复杂度，真正编码一般直接用语言或框架提供的排序能力。”

## 五、开发岗最常见的几类算法题

对你来说，最值得优先补的不是极难题，而是这几类：

### 1. 数组题

比如：

- 查找重复元素
- 两数之和
- 合并有序数组
- 删除指定元素

这些题常见考点：

- 遍历
- 双指针
- 哈希表

### 2. 字符串题

比如：

- 判断回文
- 字符统计
- 括号匹配
- 子串查找

常见考点：

- 双指针
- 栈
- 哈希表

### 3. 链表题

比如：

- 反转链表
- 找中点
- 判断环

常见考点：

- 指针移动
- 快慢指针

### 4. 树的遍历题

比如：

- 前序、中序、后序遍历
- 层序遍历
- 最大深度

这类题经常用来考：

- 递归
- 队列

### 5. 查找与排序题

比如：

- 二分查找
- 找第一个或最后一个满足条件的位置
- 排序后处理

## 六、如果面试官现场让你讲思路，怎么答

你可以按这 4 步说：

1. `先确认输入和输出`
2. `先说最直接的解法`
3. `再说能不能优化`
4. `最后补时间复杂度和空间复杂度`

这套框架非常重要，因为它会让你显得有条理。

比如一道数组题，你可以这样答：

“我会先看输入规模和是否有序，如果最直接的做法是双重循环，那复杂度可能是 O(n^2)。如果题目允许我用额外空间，我会优先考虑哈希表把查找降到更低复杂度，最后再评估空间复杂度的代价。”

这类答法对开发岗很实用。

## 七、这部分和你的开发面试怎么结合

对你来说，算法这块不需要包装成“刷题高手”，而是应该做到：

- JD 明写“数据结构和算法”时你不空
- 一面基础题你能有答题框架
- 遇到简单 coding / 白板题时不会完全卡死

也就是说，这块是：

`防守型关键模块`

不是你最强的卖点，但必须守住。

## 八、这节最值得先记住的 10 句话

1. 时间复杂度看运行时间随数据规模怎么增长
2. 空间复杂度看额外占用了多少空间
3. 数组适合按索引快速访问
4. 链表适合节点插入删除
5. 栈是后进先出，队列是先进先出
6. 哈希表非常适合做快速查找、去重和频次统计
7. 树适合表达层级关系和遍历问题
8. 双指针、哈希优化、二分查找是非常高频的算法思路
9. 回答算法题时，先讲暴力解，再讲优化解，会显得更有条理
10. 对你来说，算法的目标是“够用且不失守”，不是短时间卷成刷题型选手
