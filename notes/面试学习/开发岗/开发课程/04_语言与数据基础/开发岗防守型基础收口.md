# 开发岗防守型基础收口

用途：面试前最后一轮收口，防止基础题失守。覆盖重点：

- `Java 基础`
- `Spring Boot / Spring MVC / REST API`
- `SQL / MySQL / Redis`
- `Linux / Windows Server / IaC`
- `算法与数据结构`

---

## 一、先记住：你这条开发线的正确定位

你现在最好的定位不是：

- 纯八股 Java 后端
- 纯互联网高并发工程师
- 纯前端工程师

而是：

`有资管 / 理财业务背景、做过真实系统建设和交付、并且在后端基础题上不露怯的金融开发候选人。`

复习目标不是“什么都讲很深”，而是：

- 讲得准
- 讲得稳
- 能和项目挂上

---

## 二、Java 基础你至少要守住什么

### 1. 面向对象基本概念

你至少要答稳：

- 基本类型和引用类型
- `==` 和 `equals`
- 接口和抽象类
- 重载和重写

### 2. 常见类和集合

你至少要答稳：

- `String / StringBuilder / StringBuffer`
- `List / Set / Map`
- `ArrayList / LinkedList`
- `HashMap`
- 为什么重写 `equals` 往往也要重写 `hashCode`

### 3. 并发基础

你至少要答稳：

- 什么是线程
- 什么是线程安全
- `synchronized`
- `volatile`
- 为什么 `HashMap` 不是线程安全的

### 4. 你在 Java 这块的目标

不是讲源码，而是做到：

`常见高频题能正面答，项目里能把 Java 和业务逻辑挂起来。`

---

## 三、后端系统骨架你至少要守住什么

### 1. Spring Boot

你要答稳：

- Spring Boot 是什么
- 为什么企业后端常用它
- 它在你的项目里承接什么

最关键的是：

`你项目里的 Spring Boot 不是框架名，而是承接业务规则和接口能力的后端服务。`

### 2. Spring MVC / REST API

你要答稳：

- 请求怎么进后端
- `Controller -> Service -> Repository` 这条链路
- 参数校验为什么重要
- 为什么统一返回结构有价值

### 3. SQL / 查询功能

你要答稳：

- SQL 是做什么的
- 表结构设计大概怎么理解
- 表结构变更和历史数据回填是什么意思
- 查询功能开发不只是写 `select`

---

## 四、数据库和缓存你至少要守住什么

### 1. MySQL

你要答稳：

- MySQL 是关系型数据库
- 适合核心结构化业务数据
- 索引是什么
- 事务是什么

### 2. Redis

你要答稳：

- Redis 是高性能内存型键值存储
- 常见用途是缓存
- 为什么系统里常同时用 MySQL 和 Redis

### 3. 缓存高频概念

你至少要知道：

- 缓存穿透
- 缓存击穿
- 缓存雪崩

你现在不一定要讲很深，但不能完全陌生。

---

## 五、环境和交付链路你至少要守住什么

### 1. Linux

你要答稳：

- Linux 是常见服务器环境
- 服务运行、日志排查、脚本执行经常在 Linux 上发生

### 2. Windows Server

你要答稳：

- 它也是企业环境里的服务器系统
- 你不一定深度运维，但知道混合环境存在

### 3. IaC

你要答稳：

- IaC 是基础设施即代码
- 核心价值是标准化、可复现和自动化
- Ansible 更偏配置和批量执行
- Terraform 更偏资源定义和编排

### 4. CI/CD / Git / Jenkins

你要答稳：

- 代码提交、构建、制品、发布这条链
- 你们公司的流程更像 `Continuous Delivery`
- Jenkins Pipeline 是统一流程入口
- Git 是版本控制和团队协作工具

---

## 六、算法和数据结构你至少要守住什么

这块你不用卷很深，但至少不能完全空白。

你要答稳：

- 时间复杂度 / 空间复杂度
- 数组、链表、栈、队列、哈希表、树的基本适用场景
- 双指针、哈希优化、二分查找这些常见思路

对你来说，这块定位是：

`防守型模块，不是主秀场。`

---

## 七、如何把这些基础和你的项目挂起来

这是最重要的一层。

### 产品生命周期管理系统

挂这些点：

- Spring Boot
- React
- REST API
- Controller / Service 分层
- 业务规则系统化

### 广发证券授信变动查询

挂这些点：

- SQL
- 查询功能
- 表结构变更
- 历史数据回填
- CSV 导出

### TA 自动化测试

挂这些点：

- Jenkins Pipeline
- Oracle / SQL
- Linux 脚本
- 文件交互
- 批处理验证

### TA 批前预警稽核平台

挂这些点：

- 规则平台化
- 风险前置
- 需求梳理
- 业务检查和参数校验

---

## 八、如果面试官问：你觉得自己开发基础的强弱项分别是什么

你可以这样答：

“我现在的强项更偏真实金融系统项目经验、业务规则系统化、接口联调、批处理相关场景以及交付链路理解。基础技术上，我能比较稳地讲 Java 常见高频题、Spring Boot 后端系统、SQL/MySQL/Redis 这类问题。相对来说，我不是那种纯互联网高并发方向的工程师，算法和基础设施平台能力是我保持防守型准备的部分，但我的优势是能把技术放回真实资管业务系统里去讲。”

这句非常适合你。

---

## 九、面试前最后一轮应该怎么复习

建议按这个顺序：

1. 先复习核心项目
2. 再复习项目绑定型技术骨架
3. 再过一遍这份防守型基础收口
4. 最后看英语和金融业务补充

不要反过来一上来先刷很多基础题。

---

## 十、你现在最该记住的 12 句话

1. 你的最佳定位是懂金融系统的开发候选人
2. Java 要求是高频题不露怯，不是源码专家
3. Spring Boot / REST API 要和项目一起讲
4. Controller / Service / Repository 这条链要讲顺
5. SQL / 查询功能 / 表结构变更 / 回填是中后台基本功
6. MySQL 讲核心数据，Redis 讲缓存和高频访问
7. Linux / Windows / IaC 讲环境认知，不装运维专家
8. CI/CD / Jenkins / Git 讲交付链路，不讲平台建设者姿态
9. 算法是防守模块，不是主战场
10. 所有技术基础最好都挂回项目
11. 面试前复习顺序应该是项目优先，技术收口其次
12. 你最大的优势仍然是真实资管业务系统经验
