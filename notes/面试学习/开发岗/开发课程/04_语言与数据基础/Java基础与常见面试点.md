# Java 基础与常见面试点

这份笔记的目标不是让你把 Java 所有底层都背下来，而是让你在开发岗面试里：

- 能把 Java 的基础概念讲清楚
- 能回答最常见的一批高频题
- 能把这些基础题和你自己的项目联系起来

## 一、先讲一句话定位

对你来说，Java 在这条开发岗主线里不是“锦上添花”，而是：

`主语言主战场。`

所以这部分和 React 不一样。

- React 的目标是“加分、不失分”
- Java 的目标是“正面能答题”

## 二、Java 最简单是什么

如果面试官问你“Java 是什么”，最稳的答法不是背百科，而是这样讲：

“Java 是一种面向对象、跨平台的编程语言，企业级应用里用得很多，尤其在后端系统和中大型业务系统建设里比较常见。对我来说，它最大的特点是生态成熟、工程化能力强，比较适合做像金融业务系统这种稳定性要求高的应用。”

这里面你至少要听懂 3 个词：

- `面向对象`
- `跨平台`
- `生态成熟`

### 1. 面向对象是什么意思

最简单地说：

`不是围绕一堆零散函数写程序，而是围绕对象、属性、行为来组织代码。`

比如在业务系统里，你很容易想到：

- 产品
- 客户
- 交易申请
- 确认结果

这些都很适合抽象成对象。

### 2. 跨平台是什么意思

Java 代码不是直接跑在具体操作系统上的，而是先编译成字节码，再交给 JVM 去执行。

所以常见说法是：

`一次编写，到处运行`

当然这句话不能绝对化理解，但在面试里这样说是没问题的。

### 3. 为什么说生态成熟

因为 Java 在企业里积累了很多成熟组件：

- Spring
- Maven
- Tomcat
- 各类数据库驱动
- 各类中间件整合方案

这也是为什么很多金融系统、政府系统、大型企业应用会继续大量用 Java。

## 三、Java 基础里最容易被问的 8 个点

## 1. 基本类型和引用类型

### 基本类型

最常见有：

- `int`
- `long`
- `double`
- `boolean`
- `char`

它们存的是值本身。

### 引用类型

比如：

- `String`
- 数组
- 自定义对象
- 集合

它们本质上操作的是对象引用。

你在面试里最稳的讲法是：

“Java 里基本类型保存的是具体值，而引用类型保存的是对象的引用。像 int、long 这些属于基本类型，String、数组、集合和自定义类对象一般都属于引用类型。”

## 2. == 和 equals 的区别

这是 Java 面试高频中的高频。

最简单记法：

- `==`：比较是不是同一个
- `equals`：比较内容是不是相等

但你要知道这句话不够完整。

更准确地说：

- 基本类型用 `==` 比较的是值
- 引用类型用 `==` 比较的是是否指向同一个对象
- `equals` 默认继承自 `Object`，很多类会重写它来比较内容

比如 `String` 就重写了 `equals`。

面试答法：

“在 Java 里，基本类型用 == 比较的是值；如果是对象引用，== 比较的是两个引用是不是指向同一个对象。equals 更常用于比较内容是否相等，不过它最终取决于类有没有重写 equals，比如 String 就重写了这个方法来比较字符串内容。”

## 3. String、StringBuilder、StringBuffer 的区别

这题非常常见。

### String

- 不可变
- 每次拼接本质上可能产生新对象

### StringBuilder

- 可变
- 线程不安全
- 单线程字符串拼接更常用

### StringBuffer

- 可变
- 线程安全
- 但性能通常比 StringBuilder 低一些

最稳答法：

“String 是不可变对象，频繁拼接字符串时会有额外开销；StringBuilder 是可变的，适合单线程场景下高频拼接；StringBuffer 也是可变的，但做了线程安全处理，所以开销会更大一些。”

## 4. 接口和抽象类的区别

这题也很常见。

最简单理解：

- `接口` 更像能力约定
- `抽象类` 更像一组有共性的基类

你可以这样答：

“接口更强调行为规范和能力定义，适合做解耦和扩展；抽象类更适合抽取多个类之间的共性，可以包含成员变量和部分公共实现。简单理解，接口更像‘你要具备什么能力’，抽象类更像‘你本质上是什么类型的一类对象’。”

## 5. 重载和重写

### 重载 Overload

- 同一个类里
- 方法名相同
- 参数列表不同

### 重写 Override

- 子类对父类方法重新实现
- 方法签名保持兼容

面试答法：

“重载发生在同一个类内部，方法名相同但参数不同；重写发生在继承关系中，子类对父类已有方法重新实现。”

## 6. 异常 Exception

你至少要知道两类：

- `Checked Exception`
- `Unchecked Exception`

常见理解是：

- 编译器要求显式处理的，通常叫受检异常
- 运行时异常一般属于非受检异常

面试不用讲太深，先会这版：

“Java 异常体系里，常见会区分受检异常和运行时异常。受检异常通常要求显式处理，运行时异常更多和程序逻辑问题有关，比如空指针、数组越界这类。”

## 7. 集合 Collection

这块后面还值得单独补，但现在你至少要先知道最基本的关系：

- `List`：有序，可重复
- `Set`：不允许重复
- `Map`：键值对

常见实现：

- `ArrayList`
- `LinkedList`
- `HashSet`
- `HashMap`

如果面试官先问很基础的：

“List、Set、Map 有什么区别？”

你可以答：

“List 是有序可重复集合，Set 主要用于去重，Map 存的是 key-value 结构。实际开发里最常见的是 ArrayList 和 HashMap。”

## 8. 多线程 / 并发

这块你后面还需要系统补，但现在至少先建立一个基础认知：

- 线程是程序执行的一个基本单位
- 并发编程的重点不只是“多开几个线程”，而是：
  - 线程安全
  - 共享资源竞争
  - 同步与锁

你现在先不用讲得很深，但要知道：

“多线程带来的核心问题通常不是语法，而是共享数据访问时的一致性和线程安全。”

1. Java 是面向对象、跨平台、生态成熟的企业级语言
2. 基本类型存值本身，引用类型存对象引用
3. 基本类型的 `==` 比较值，对象的 `==` 比较引用

## 七、集合基础：List、Set、Map

这组题在开发岗面试里非常高频，因为它和日常业务开发直接相关。

## 1. List、Set、Map 最简单的区别

### List

- 有序
- 允许重复

最适合的理解是：

`一组按顺序排列的数据。`

比如：

- 查询结果列表
- 页面表格数据
- 任务执行顺序

### Set

- 不允许重复
- 一般用于去重

最适合的理解是：

`一组不希望出现重复元素的数据。`

比如：

- 去重后的产品代码集合
- 去重后的用户 ID 集合

### Map

- 键值对结构
- 通过 key 找 value

最适合的理解是：

`一组 key-value 映射关系。`

比如：

- `产品代码 -> 产品信息`
- `字段名 -> 字段值`

面试最稳答法：

“List、Set、Map 是 Java 里最基础的三类集合。List 有序且允许重复，适合表示一组顺序数据；Set 主要用于去重；Map 保存的是 key-value 结构，适合做映射关系和按键快速查找。”

## 2. ArrayList 和 LinkedList

这是 List 里最常被问的一题。

### ArrayList

底层更接近：

`动态数组`

特点：

- 按索引取值快
- 尾部追加通常比较高效
- 中间插入、删除时可能要移动元素

### LinkedList

底层更接近：

`链表`

特点：

- 插入、删除节点在某些场景下更灵活
- 但按索引访问不如 ArrayList 直接

### 面试里怎么讲更稳

不要硬背大 O 复杂度就完了，你更适合先这样讲：

“ArrayList 底层更接近动态数组，所以随机访问更方便；LinkedList 底层更接近链表，在插入删除方面更灵活，但按索引访问通常不如 ArrayList 直接。实际业务开发里，如果主要是存查询结果、列表数据，大多数场景更常用 ArrayList。”

这最后一句很重要，因为它体现出你有实际开发感，不只是背定义。

## 3. HashMap 是什么

这是 Map 里最常见的一题。

你先把它理解成：

`最常用的键值对集合。`

它最典型的用途是：

- 通过 key 快速找 value
- 在业务代码里做映射、缓存、索引

比如：

- 根据产品代码找到产品对象
- 根据字段名找到字段值

### 你现在至少要知道这几件事

1. `HashMap` 允许一个 `null key`
2. `HashMap` 允许多个 `null value`
3. 它不是线程安全的
4. 它和 `hashCode`、`equals`` 有直接关系

## 4. 为什么重写 equals 往往也要重写 hashCode

这是面试里很喜欢追问的一点。

最简单的理解是：

如果两个对象你认为“内容相等”，那它们参与哈希相关集合时，也应该表现一致。

比如在：

- `HashMap`
- `HashSet`

这些集合里，底层会用到哈希值来帮助定位。

所以如果你只重写了 `equals`，没重写 `hashCode`，就可能出现：

- 逻辑上两个对象你觉得相等
- 但放进 `HashSet` 里行为不一致
- 或者当作 `HashMap key` 时查找异常

面试最稳答法：

“因为在 HashMap、HashSet 这类基于哈希的集合里，hashCode 和 equals 是一起工作的。如果两个对象通过 equals 判断是相等的，那它们的 hashCode 也应该一致，否则在哈希集合中可能出现查找、去重行为不符合预期的情况。”

## 5. List、Set、Map 和你项目的关系怎么讲

这部分你不用硬讲源码，但可以这样理解：

### List

最像：

- 查询结果列表
- 页面表格返回数据
- 批量任务结果集合

### Set

最像：

- 去重后的机构列表
- 去重后的字段集合

### Map

最像：

- 字段名到字段值的映射
- 产品代码到产品对象的映射
- 某类配置项的键值组织

如果你面试里能把集合和这些真实业务场景连起来，会显得你不是只会背八股。

## 6. 这一组最值得你先记住的 6 句话

1. List 有序可重复，Set 主要用于去重，Map 保存 key-value 映射
2. ArrayList 更适合大多数普通列表场景
3. LinkedList 更像链式结构，按索引访问通常不如 ArrayList
4. HashMap 是最常见的键值对集合
5. HashMap 不是线程安全的
6. 重写 equals 时，通常也要重写 hashCode

## 八、并发基础：线程、线程安全、synchronized、volatile

这组题会比前面稍微技术一点，但也是开发岗非常常见的基础题。

## 1. 线程是什么

最简单理解：

`线程是程序执行的一个基本单位。`

如果你先只记业务化一点的理解，可以记：

- 一个程序里可以同时做多件事
- 这些“同时做事”的执行单元，就可以理解成线程

比如一个系统里可能同时发生：

- 用户请求处理
- 定时任务执行
- 文件读取
- 消息消费

所以线程的核心不是语法，而是：

`让程序具备并发处理能力。`

## 2. 并发最核心的问题是什么

不是“怎么创建线程”，而是：

- 多个线程会不会同时修改同一份数据
- 修改顺序会不会乱
- 一个线程改了，另一个线程能不能立刻看见

所以并发最核心的 3 个关键词你先记住：

1. `线程安全`
2. `可见性`
3. `有序性`

现在你先重点记第一个就够。

## 3. 什么叫线程安全

最简单理解：

`多个线程同时访问一段代码或一份数据时，结果仍然正确、可控。`

如果一段代码在单线程下没问题，但多线程一跑就出错、数据乱了、结果不稳定，那它就不是线程安全的。

### 为什么会不安全

最常见原因是：

- 多个线程同时修改共享变量
- 操作不是原子的
- 一个线程改了，另一个线程没及时看到

## 4. 为什么 HashMap 不是线程安全的

你前面已经记住了这个结论，现在补上更像面试的理解。

因为 `HashMap` 在设计上默认就是给普通单线程场景用的。

如果多个线程同时对一个 `HashMap` 做：

- put
- remove
- resize

就可能出现：

- 数据覆盖
- 结果不一致
- 结构异常

所以面试里最稳的说法是：

“HashMap 默认不是线程安全的，因为多个线程同时修改它时，可能导致数据不一致或结构问题。如果在并发场景里需要线程安全的映射结构，一般会考虑 ConcurrentHashMap 这类并发集合。”

这句话很够用了。

## 5. synchronized 是什么

最简单理解：

`给一段代码或一个方法加锁，让同一时刻只有一个线程进去执行。`

它的作用就是：

`控制并发访问，避免多个线程同时改共享资源。`

### 你可以怎么理解它

如果一段逻辑涉及：

- 改共享变量
- 改共享集合
- 改共享状态

那就可能需要同步控制。

`synchronized` 就是 Java 里最经典的同步手段之一。

### 面试最稳答法

“synchronized 是 Java 里常见的同步关键字，可以给方法或代码块加锁，保证同一时刻只有一个线程执行这段受保护的逻辑，常用于控制对共享资源的并发访问。”

## 6. volatile 是什么

这个词经常会和 `synchronized` 一起问。

你先记住最简单的区别：

- `synchronized` 更强调互斥和同步
- `volatile` 更强调可见性

### 什么叫可见性

最简单理解就是：

`一个线程修改了变量，其他线程能更快看到最新值。`

如果没有 `volatile`，某个线程可能还在用旧值。

### 但 volatile 不等于线程安全

这是面试里很爱考的一点。

`volatile` 能解决的是“看得见”，
但不一定能解决“多个线程一起改时会不会冲突”。

所以你可以这样理解：

- 适合做状态标记
- 不适合直接替代所有同步控制

### 面试最稳答法

“volatile 主要用于保证变量在多线程场景下的可见性，也就是一个线程修改后，其他线程能及时看到最新值。但它不等于完整的线程安全控制，因为它不能替代对复合操作的互斥保护。”

## 7. synchronized 和 volatile 的区别

你先记这版最够用的：

- `synchronized`：控制同一时刻只能一个线程执行，解决互斥和同步问题
- `volatile`：保证变量修改后的可见性，但不负责完整互斥

面试答法：

“synchronized 更偏向加锁和互斥控制，适合保护临界区代码；volatile 更偏向保证变量可见性，适合状态标记这类简单场景。前者更强，后者更轻，但解决的问题不完全一样。”

## 8. 这一组你先记住的 6 句话

1. 线程是程序执行的基本单位
2. 并发最核心的问题是线程安全
3. 线程安全指多个线程同时访问时结果仍然正确可控
4. HashMap 默认不是线程安全的
5. synchronized 用来做加锁和同步控制
6. volatile 主要解决可见性，不等于完整线程安全
