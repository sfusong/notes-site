# Java 手撕算法冲刺

这个目录只服务一个目标：

`高盛工银理财 software developer onsite 里 30 分钟 Java coding / 算法题不露怯。`

## 复习定位

- 面向金融机构 software developer onsite
- 题目优先级：基础高频 > Java 语法复健 > 手写稳定 > 能讲清复杂度
- 不按大厂长线刷题路线发散
- 暂时不碰难题、怪题、复杂动态规划、复杂图论
- 重点训练：数组、字符串、哈希表、双指针、栈、链表、二分、基础树
- 顺手复习：`for` 循环、数组、`String`、`char`、`HashMap`、`ArrayList`、`Deque`、`Queue`、递归、空值判断、返回数组

你只需要记住这些基础定位：

- 时间复杂度看运行时间随数据规模怎么增长，常见是 `O(1)`、`O(log n)`、`O(n)`、`O(n log n)`、`O(n^2)`。
- 空间复杂度看额外用了多少空间，比如哈希表、数组、递归栈。
- 数组适合下标访问和顺序遍历。
- 哈希表适合查找、去重、统计频次、建立映射。
- 栈适合括号匹配和回退类问题。
- 队列适合 BFS 和按顺序处理任务。
- 双指针适合数组、字符串和链表问题。
- 二分查找的前提通常是有序或具备单调性。
- 链表题重点是指针关系，树题重点是递归和队列遍历。

## 为什么是基础 12 题

不是因为 12 题能覆盖所有算法，而是因为你现在只有几天，目标也不是变成刷题型候选人。

12 题的作用是：

- 覆盖 onsite 最可能出现的基础题型
- 每题都能在 3 到 8 分钟内手写完
- 反复练 Java 基础语法和常用集合
- 适合 30 分钟连续做 3 题的节奏
- 数量足够小，能真正练熟，而不是看过就忘

这 12 题更像一个“最小可用题库”。先把它们写顺，再根据时间加同类型变体。

复审后的结论是：12 题作为主干够用，但需要补几个真正的语法缺口：树递归遍历、`HashSet`、字符串双指针、递归返回值。现在统一整理在 01 主文件里。

## 现场答题顺序

1. 先复述题意，确认输入输出和边界条件。
2. 给一个最直接思路，再说如何优化。
3. 写 Java 代码时优先保证完整、可读、无低级语法错误。
4. 写完立刻口头跑一个例子。
5. 最后补时间复杂度和空间复杂度。

## 现场口头模板

“我先确认一下题意。这个题的核心是……。最直接可以用……，复杂度是……。这里我会用……来优化。代码里我会注意空数组、重复元素、边界下标这些情况。最后时间复杂度是……，空间复杂度是……。”
