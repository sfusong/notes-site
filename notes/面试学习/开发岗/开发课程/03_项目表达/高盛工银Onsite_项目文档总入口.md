## 2. 三条主项目怎么定位

| 项目 | 面试定位 | 最适合证明 |
|---|---|---|
| 产品生命周期管理系统 | 自研前后端系统，承接特殊产品节点提醒 | Spring Boot、React、数据库设计、数据中台联调、邮件/OA 接口 |
| TA 自动化测试 | 围绕申请文件、批处理、确认文件、数据库对账的自动化回归 | Java 文件解析、Jenkins Pipeline、SQL 对账、批处理理解 |
| TA 静态参数稽核 | 批前检查关键配置，提前暴露运营和清算风险 | 规则系统化、批前风控、参数检查、异常汇总 |

## 3. 每条项目最低背诵标准

### 产品生命周期管理系统

1. 这是正式产品管理系统上线前的自研辅助系统。
2. 它解决目标盈、分红封闭式、定期分红等特殊产品的节点提醒问题。
3. 后端是 Spring Boot，前端基于内部 React 平台。
4. 你主做后端规则判断、数据库设计、数据中台联调、邮件/OA 接口。
5. 最适合展开 `目标盈提醒`。
6. 追问重点：为什么自研、表怎么设计、目标盈怎么判断、提醒错了怎么排查。

如果面试官继续问实现细节，转到：

- [产品生命周期管理系统复现项目](notes/面试学习/开发岗/开发课程/03_项目表达/产品生命周期管理系统复现项目/README.md)

### TA 自动化测试

1. 它解决 TA 版本升级、代销联调和回归测试效率问题。
2. 核心闭环是：申请文件生成 -> TA 批处理 -> 确认文件输出 -> 数据库对账。
3. 你参与技术选型、确认文件解析、字段级比对和数据库对账。
4. Jenkins 负责统一触发，SQL 负责环境准备和预期结果，Java 负责文件解析和比对。
5. 最适合展开 `确认文件内容自动校验`。
6. 追问重点：你写了什么、expected/actual 怎么比、Jenkins 怎么串、错了怎么定位。

### TA 静态参数稽核

1. 它是 TA 批前预警里偏自研的一部分。
2. 它和交易类预警不同，更偏参数漏设、错设、状态冲突检查。
3. 典型检查项包括托管账号、销售商清算账号、费率、开放期、产品状态、日期。
4. 你的参与重点是规则清单、口径设计、检查项系统化和落地推进。
5. 它的价值是把人工经验沉淀成批前可重复执行的系统检查。
6. 追问重点：为什么自研、和恒生 AOP 区别、异常怎么给业务用、规则错了怎么定位。

| 项目 | 什么时候讲 | 讲到什么程度 |

## 5. React 怎么嵌入项目

React 不单独做主项目讲。

只在产品生命周期系统里这样带出：

- 前端承担表单录入、列表展示、提醒状态展示、接口联动。
- 你参与接口联动和部分页面开发，不把自己说成唯一前端负责人。
- 如果被问 React 原理，回到 [React在你的项目里怎么讲.md](notes/面试学习/开发岗/开发课程/03_项目表达/React在你的项目里怎么讲.md)。

## 6. 最后 24 小时项目复习

只做 4 件事：

1. 每条主项目背 1 分钟讲法。
2. 每条主项目准备 3 个追问。
3. TA 自动化测试补 `确认文件解析 + 数据库对账`。
4. 产品生命周期系统补 `目标盈提醒 + 表结构 + 邮件/OA 接口`。

不要再扩新项目。

一句话收束：

`主战项目讲深，备用项目讲短；开发参与讲具体，非主责部分讲边界。`
