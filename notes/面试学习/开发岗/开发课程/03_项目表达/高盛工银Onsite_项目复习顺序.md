# 高盛工银 Onsite：项目复习顺序

这份只服务一个目标：

`在高盛工银 onsite 前，把项目复习压到最短路径。`

结合当前已知信息：

- 一面偏技术和 coding
- onsite 会有 `30 分钟 Java coding`
- 后续 panel 会继续看 `项目 + behavior + 技术理解`
- 岗位本质还是 `软件开发工程师`

所以项目复习原则不是“全都背”，而是：

- `先保 coding`
- `再保 3 条最能证明开发参与度的项目`
- `最后才看加分项`

---

## 一、优先级排序

### 第一优先级

1. `产品生命周期管理系统`
2. `TA 自动化测试`

### 第二优先级

3. `TA 批前预警里的静态参数稽核`

### 第三优先级

4. `Apprentice`

---

## 二、为什么这样排

## 1. 产品生命周期管理系统

这是你最像“自己确实参与开发了一套系统”的项目。

它同时覆盖：

- Spring Boot
- React
- 数据库设计
- 数据中台联调
- 邮件 / OA 提醒接口
- 业务规则落地

这条项目最适合回答：

- 你做过哪些开发工作
- 你怎么理解后端系统
- 你怎么把业务规则做成系统能力

---

## 2. TA 自动化测试

这条特别贴岗位，因为岗位本身就强调：

- 测试上线
- 自动化
- SDLC
- DevOps 会涉及一些

这条项目最适合回答：

- 你怎么理解自动化测试
- 你是否参与过技术实现
- 你怎么理解 Jenkins / Oracle SQL / Linux / 批处理链路

---

## 3. TA 批前预警里的静态参数稽核

这条项目的价值在于：

- 规则系统化
- 风险前置
- 自研检查能力

但它相对没有前两条那么强“开发工程师味”，所以放在第二优先级。

它更适合回答：

- 你怎么理解金融系统里的风险控制
- 你怎么把规则转成系统检查项
- 你是否参与过自研检查能力建设

---

## 4. Apprentice

这条只作为：

- 个人项目
- 学习能力
- 对新技术的主动尝试

不要把它放在主战场。

只有当面试官主动问：

- 你最近有没有自己做过项目
- 你怎么看 AI / 新技术
- 你有没有额外的 coding 兴趣

再拿出来。

---

## 三、每条项目复习到什么程度

## 1. 产品生命周期管理系统

你至少要能稳定答这 6 句：

1. 这是正式产品管理系统上线前的自研辅助系统。
2. 它解决的是特殊产品生命周期节点管理和提醒问题。
3. 技术上是内网前后端系统：后端 Spring Boot，前端基于内部 React 平台。
4. 你主做的是后端：规则判断、数据库设计、数据中台联调、邮件/OA 接口。
5. 最适合展开的功能是 `目标盈提醒`。
6. 它最能证明你确实参与过系统开发，而不是只做需求和交付。

你至少要能扛的追问：

- 为什么要自研
- 目标盈提醒怎么实现
- 数据库大概怎么设计
- 提醒错了怎么排查

---

## 2. TA 自动化测试

你至少要能稳定答这 6 句：

1. 它解决的是 TA 版本升级和代销联调中的回归测试效率问题。
2. 核心闭环是：申请文件进入 -> 批处理 -> 确认文件输出 -> 数据库比对。
3. 它降低了对外部代销联调和人工核对的依赖。
4. 你参与了思路提出、技术选型、确认文件解析、数据库比对这部分实现。
5. 工作流主要通过 Jenkins Pipeline 串起来，底层会涉及 Oracle SQL、Linux、批处理。
6. 这条项目最能证明你理解自动化测试、文件解析、数据比对和工程链路。

你至少要能扛的追问：

- 你具体写了什么
- 确认文件怎么比对数据库
- 比对错了怎么排查
- 为什么这条链路适合做自动化

---

## 3. TA 静态参数稽核

你至少要能稳定答这 5 句：

1. 这条是 TA 批前预警里偏自研的一部分。
2. 它和交易类批前预警不同，更偏静态参数和配置正确性检查。
3. 典型检查项包括托管账号、销售商清算账号、费率、开放期、状态、日期等。
4. 你的参与重点是规则清单梳理、口径设计、检查项系统化和落地推进。
5. 它的本质是把运营控制里的人工检查沉淀成系统能力。

你至少要能扛的追问：

- 为什么这部分值得自研
- 它和恒生 AOP 的区别是什么
- 稽核结果怎么给业务使用
- 规则错了怎么定位

---

## 4. Apprentice

你只要准备这 3 句：

1. 这是你最近做的个人开发项目。
2. 它更多体现你对新技术和工具的主动学习。
3. 它不是你这次面试的主项目，只在面试官主动追问时展开。

适合展开的方向：

- 为什么做
- 你学到了什么
- 体现了什么工程意识

不适合主动展开：

- 抢过金融项目主线
- 讲太细的 AI 架构

---

## 四、你这次 onsite 的项目复习顺序

## 第一轮：只复习主战场

顺序：

1. 产品生命周期管理系统
2. TA 自动化测试
3. TA 静态参数稽核

要求：

- 每条都能讲 `1 分钟`
- 每条都能答 `3 个追问`

---

## 第二轮：补技术关键词

每条项目都顺手补这几个词：

- Spring Boot
- React
- SQL / 数据库设计
- Jenkins Pipeline
- Linux
- Git
- 接口联调
- UAT / 上线

目标：

- 项目一旦讲到技术，不会空掉

---

## 第三轮：Apprentice 只做备用

只在下面情况再讲：

- 面试官主动问个人项目
- 面试官问你最近学习什么新技术
- 面试官问你对 AI 工具的使用

---

## 五、最短复习法

如果你时间非常紧，就只做这件事：

### 先练这 3 个 1 分钟讲法

1. 产品生命周期管理系统
2. TA 自动化测试
3. TA 静态参数稽核

### 再各准备 3 个追问

#### 生命周期系统

- 为什么自研
- 目标盈提醒怎么做
- 你具体写了什么

#### TA 自动化测试

- 闭环是什么
- 你实现了哪部分
- 结果不一致怎么排查

#### 静态参数稽核

- 为什么做
- 规则检查哪些内容
- 和交易类预警区别是什么

---

## 六、一句话结论

这次 onsite 的项目复习，不是四条项目平均用力，而是：

`产品生命周期管理系统 + TA 自动化测试` 主打，`TA 静态参数稽核` 补强，`Apprentice` 备用。
