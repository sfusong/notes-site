# 开发岗课程总纲

这份文件现在是：

`开发岗学习线的唯一课程总纲。`

以后如果课程顺序继续调整，就直接改这份，不再新增“重排版”“新版”“补充版”。

也就是说：

- 想知道课程真正怎么学，看这份
- 其他 `README` 更偏资料导航
- 其他专题和笔记更偏具体内容

这份总纲只做一件事：

`明确开发岗这条线到底应该按什么顺序学。`

如果你不是按学习顺序，而是按内容类型查资料，直接看：

- 项目面试：[项目面试总索引.md](notes/面试学习/开发岗/开发课程/03_项目表达/项目面试总索引.md)
- 基础技术：[基础技术面试总索引.md](notes/面试学习/开发岗/开发课程/04_语言与数据基础/基础技术面试总索引.md)

先说明一个关键原则：

当前 `面试学习/开发/` 下面的目录，

- `01_岗位与计划`
- `02_系统与交付基础`
- `03_项目表达`
- `04_语言与数据基础`

更像是：

`资料存放方式`

不是：

`实际学习顺序`

之前的问题就在这里。

如果把“目录分类”直接当成“课程顺序”，就很容易出现：

- 刚讲完项目
- 又跳回 Java 基础
- 再跳回系统部署

这种割裂感。

所以现在开发岗这条线，统一改成按 **阶段主线** 学，而不是按目录看。

## 一、正确的课程主线

开发岗这条线，应该按下面 6 个阶段推进：

### 阶段 0：岗位定位

先回答：

- 这个岗位到底要什么
- 你为什么能投
- 你不是哪类人
- 你最该补什么

这个阶段的目标不是学知识，而是先立住：

`我是一个金融系统开发候选人，不是纯互联网后端，也不是纯产品。`

对应材料：

- [工银高盛开发岗_JD拆解.md](notes/面试学习/开发岗/开发课程/01_岗位与计划/工银高盛开发岗_JD拆解.md)

### 阶段 1：核心项目主线

这是最重要的一阶段，应该先于大部分八股。

目标是先把你最值钱的 4 条内容讲顺：

1. `产品生命周期管理系统`
2. `TA 批前预警稽核平台`
3. `TA 自动化测试`
4. `接口联调 / UAT / 上线切换`

为什么这一阶段要先学？

因为你的开发岗竞争力，最核心不是 Java 八股，而是：

- 金融业务背景真
- 系统建设经历真
- 交付链路经历真
- 规则系统化经验真

如果这几条讲不顺，后面补再多基础题，也还是容易像“会背概念的人”。

### 阶段 3：项目绑定型技术骨架

这是之前最容易乱掉的一层。

这层的重点不是“单独背 Java”，而是：

`把后端技术放回你的项目里讲。`

包括：

- Spring Boot 在你的项目里到底承接什么
- REST API 是怎么服务前后端和系统协作的
- 请求怎么进 Controller，怎么进 Service
- 查询功能、表结构变更、历史数据回填怎么讲
- React 在你的项目里怎么讲

这一阶段的目标是：

`让技术不悬空，而是变成你的项目骨架。`

## 二、为什么之前会显得乱

核心原因只有一个：

`资料目录按“内容类型”分，但教学顺序没有单独抽出来。`

于是就会出现：

- 目录是清楚的
- 课程却容易跳

最典型的问题就是：

- 刚讲完项目
- 又回去讲 Java 基础
- 然后再讲项目绑定技术

这在逻辑上当然会断。

所以现在统一原则是：

`项目主线优先，项目绑定技术其次，纯八股后置。`

## 三、接下来正确的课程顺序

后面的课程，不应该再先讲纯 Java 集合、Optional、Stream。

应该按这个顺序继续：

1. `产品生命周期管理系统高频追问怎么答`
2. `TA 批前预警稽核平台高频追问怎么答`
3. `React 在你的项目里怎么讲`
4. `广发证券授信变动查询项目怎么讲`
5. `金融开发岗英语项目表达`
6. `金融市场 / 风险管理基础认知`

然后如果再往后，才继续：

7. `Java 集合 / 泛型 / Optional / Stream`
8. `MySQL / Redis 深一点`
9. `Linux / Windows Server / IaC 稍深化`
10. `算法最小刷题集`

## 四、你现在最该记住的一句话

开发岗这条线最好的胜法不是：

`把自己训练成纯八股后端`

而是：

`把自己训练成一个能讲清金融系统项目、能讲清交付链路、能把后端技术放回项目里说、同时在基础题上不露怯的金融开发候选人。`
