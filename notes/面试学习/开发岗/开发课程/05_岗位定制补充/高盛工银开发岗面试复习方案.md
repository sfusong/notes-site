# 高盛工银开发岗面试复习方案

这份方案是基于两部分信息整理的：

1. 这次高盛工银开发岗 JD
2. 面试官 Deanna 的背景特征

从你的信息看，这场面试大概率不会只停留在纯八股层面，而会更关注：

- 你是否真的做过金融系统
- 你是否理解系统从设计到上线这条完整链路
- 你能不能把业务、系统边界、数据流转和交付说清楚
- 你英文表达是否稳定

---

## 一、先说结论：这场面试最可能看什么

如果结合 JD 和 Deanna 的履历，我建议你把复习重点排成下面 5 层：

1. `项目和系统链路`
2. `Java / Spring Boot / API / 数据库`
3. `系统运行与交付`
4. `金融业务理解`
5. `英文表达稳定性`

这场面试不太像那种只刷算法题的开发岗，更像：

`看你是不是一个能进入金融机构真实系统建设语境的人。`

---

## 二、结合 JD，看面试官最可能关注的能力

### 1. 与业务紧密合作

JD 明写要和不同部门的业务方紧密合作，为前中后台提供方案。

这意味着你要准备好回答：

- 你怎么跟业务确认口径
- 你怎么把业务规则翻译成系统逻辑
- 你怎么处理业务、测试、供应商之间的信息差

你的可用材料：

- 产品生命周期管理系统
- TA 批前预警稽核平台
- 接口联调 / UAT / 上线切换

### 2. 从设计、开发到部署的关键业务系统解决方案

这意味着她可能不只问：

- 你写了什么代码

还会问：

- 这个系统为什么要这样设计
- 数据或请求是怎么流转的
- 上线和切换怎么控制风险
- 你理解的系统边界是什么

你的可用材料：

- 产品生命周期管理系统
- TA 自动化测试
- 接口联调 / UAT / 上线切换

### 3. 学习和利用不同技术栈

这意味着她很可能接受“不是单一技术栈出身”的候选人，但会看你是否真的理解系统工程。

对你来说是加分点，因为你能讲：

- Java / Spring Boot
- Python
- React
- SQL
- Linux
- 自动化和交付

但讲法要统一成：

`我不是堆技术栈，而是在真实项目中按问题选择技术。`

### 4. Linux / Windows Server / Nginx / Tomcat / Redis / MySQL / CI/CD / IaC / Git

这些题她未必全都深问，但会通过几个点判断你是不是只会业务词。

所以你要做到：

- Java / Spring Boot 能讲项目里的真实用法
- Tomcat / Nginx 能讲系统运行关系
- Git / SDLC / CI/CD 能讲真实发布流程
- Linux 能讲日志、进程、脚本、部署场景
- IaC 至少有“认知级”说法，不至于完全空白

## 三、结合 Deanna 的经历，推测她的提问风格

从你给的信息看，Deanna 有这些明显特征：

- 在 Goldman Sachs 工程体系做了很久
- 在纽约、东京、香港、上海都工作过
- 接触过 internal / external data
- 熟悉 on-prem 架构
- 也做过 cloud native architecture in China

这类面试官通常更在意：

### 1. 你是否能讲清系统而不是只讲页面

她大概率会更喜欢听你说：

- system boundary
- data flow
- integration points
- operational control
- production rollout

而不是只说：

- 我做了几个页面
- 我会哪些语法

### 2. 你是否理解企业系统的复杂度

她可能会通过提问判断你是否理解：

- internal system vs external institution
- request path vs batch path
- development vs production
- business rule vs technical implementation

### 3. 你有没有“交付成熟度”

她大概率会喜欢能回答这些的人：

- 知道联调为什么难
- 知道上线切换为什么要有回滚预案
- 知道数据口径和系统结果为什么会不一致
- 知道金融系统里很多价值来自稳定性和控制，而不是花哨框架

## 七、你这场面试的答题风格建议

### 1. 多说系统关系，少说空泛结论

比如不要只说：

- I used Spring Boot.

更好的是：

- I used Spring Boot to implement business rules, expose APIs and support the backend service layer in a rule-driven internal financial system.

### 2. 多说 why 和 trade-off，少说工具名罗列

比如不要只说：

- We used CI/CD.

更好的是：

- In our environment, CI/CD was valuable because it standardized the path from code submission to build artifact generation and controlled production rollout.

### 3. 不要急着装特别资深，先显得稳定、真实

这类面试官通常不怕你说：

- my role was mainly...
- I was involved in...
- from my understanding...

反而更怕你讲得过头然后经不起追问。
