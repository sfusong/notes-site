# 第14课：基金的投资交易与结算

> 大纲编号：14.1.a / 14.1.b / 14.2.a / 14.3.a / 14.3.b / 14.4.a / 14.4.b / 14.4.c / 14.4.d
> 本课核心问题：基金买卖股票债券之后，资金和证券如何真正交收？

---

## 核心逻辑：交易成交不等于结算完成，基金后台还要完成清算、交收和对账

投资交易有两个层次：

```text
交易：达成买卖
结算：钱和证券真正交割
```

很多风险并不发生在“是否成交”，而发生在成交之后：

- 对手方是否履约；
- 资金是否到账；
- 证券是否交割；
- 估值和会计是否正确；
- 托管人和管理人账务是否一致。

---

## 一、证券交易和结算基础（14.1.a）｜理解

交易完成后，需要清算和交收。

| 环节 | 含义 |
|---|---|
| 交易 | 买卖双方达成成交 |
| 清算 | 计算应收应付金额和证券数量 |
| 交收 | 实际完成资金和证券转移 |

例子：基金买入股票，交易系统显示成交，只说明买卖达成；真正完成还需要资金划出、股票划入、登记过户。

---

## 二、净额结算和全额结算（14.1.b）｜掌握

### 2.1 全额结算

每笔交易单独结算。

优点：关系清楚。
缺点：资金和证券占用大，效率低。

### 2.2 净额结算

把同一参与方多笔应收应付轧差，只结算净额。

例子：

- A 应付 B 100 万；
- B 应付 A 80 万；
- 净额结算后 A 只需付 B 20 万。

净额结算可以提高效率、减少资金占用，但需要完善的中央清算和风险控制机制。

---

## 三、货银对付 DVP（14.2.a）｜掌握

DVP 是 Delivery Versus Payment，货银对付。

含义：

> 证券交付和资金支付同步进行，防止一方交了证券却收不到钱，或付了钱却收不到证券。

DVP 降低本金风险，是证券结算的重要机制。

如果没有 DVP，可能出现：

- 买方付款但卖方不交证券；
- 卖方交证券但买方不付款。

---

## 四、中央对手方 CCP｜理解

中央对手方介入后，成为所有买方的卖方、所有卖方的买方。

作用：

| 作用 | 说明 |
|---|---|
| 降低对手方风险 | 交易对手从多个机构变成中央对手方 |
| 提高清算效率 | 统一净额清算 |
| 风险管理 | 收保证金、建立违约处理机制 |

但 CCP 也会集中风险，因此需要严格风险管理。

---

## 五、基金投资交易的托管和划款（14.3.a）｜掌握

基金资产由托管人保管。基金管理人做出投资指令后，托管人要复核指令是否合规。

流程：

```text
基金经理下达投资决策
  -> 交易员执行交易
  -> 成交数据回传
  -> 管理人发送划款/交收指令
  -> 托管人复核
  -> 资金和证券交收
  -> 对账和估值
```

托管人如果发现指令违反基金合同或法规，应拒绝执行并报告。

---

## 六、银行间债券市场结算（14.3.b）｜了解

债券交易常涉及银行间市场。

银行间市场特点：

- 机构投资者为主；
- 交易方式更偏询价和做市；
- 债券品种丰富；
- 结算和托管体系与交易所不同。

基金投资债券时，需要关注：

- 交易对手；
- 结算方式；
- 券款对付；
- 交收日期；
- 债券托管账户；
- 估值和应计利息。

---

## 七、境外市场交易与结算（14.4.a / 14.4.b / 14.4.c / 14.4.d）｜了解

基金投资境外资产时，交易和结算会更复杂。

需要关注：

- 不同市场的交易日和节假日；
- 结算周期差异；
- 汇率折算和外汇资金划转；
- 境外托管和次托管安排；
- 税费、交易规则和信息披露差异；
- 跨时区导致的估值时点差异。

考试不要求你掌握每个国家规则，但要记住：境外投资不仅多了市场风险和汇率风险，也多了结算、托管、税务和时差带来的运营复杂度。

---

## 八、结算风险

常见结算风险：

| 风险 | 含义 |
|---|---|
| 本金风险 | 交付资产后未收到对价 |
| 对手方风险 | 对手不能履约 |
| 流动性风险 | 临时缺资金或缺券 |
| 操作风险 | 指令错误、系统故障、对账失败 |
| 法律风险 | 合同或规则不完善导致争议 |

DVP、净额结算、保证金、违约基金、托管复核都是为了降低这些风险。

---

## 九、高频易错点

| 易错点 | 正确理解 |
|---|---|
| 成交就等于结算完成 | 错。成交后还要清算和交收 |
| 清算和交收是一回事 | 错。清算算账，交收交钱交券 |
| DVP 增加本金风险 | 错。DVP 降低本金风险 |
| 托管人只是被动划款 | 错。托管人要复核合规性 |
| 净额结算比全额结算资金占用更高 | 错。净额结算通常降低资金占用 |
| 中央对手方没有风险管理要求 | 错。CCP 风险集中，更需要严格管理 |

## 十、典型例题

**例题1·单选**：证券结算中的 DVP 指的是：

A. 分红再投资
B. 货银对付
C. 净值披露
D. 申购确认

**答案：B**
解析：DVP 是证券交付和资金支付同步完成的机制。

**例题2·判断**：清算是计算应收应付，交收是实际完成资金和证券转移。（ ）

**答案：对。**

## 十一、考场判断框架：交易、清算、交收、托管怎么分

这章看似后台，但考试很喜欢考概念边界。

| 概念 | 一句话 | 你可以这样记 |
|---|---|---|
| 交易 | 买卖双方达成成交 | “成交了” |
| 清算 | 计算钱和券该收该付多少 | “算清楚” |
| 交收 | 真正完成钱券转移 | “交到位” |
| 托管 | 托管人保管资产并复核指令 | “看住钱和券” |

### 两个特别容易错的点

第一，成交不等于安全结束。成交后如果对手方、资金划付、证券交割、系统处理出现问题，仍然会产生结算风险。

第二，托管人不是简单“听管理人指挥划款”。托管人要复核投资指令是否符合法规和基金合同，如果发现违规，应当拒绝执行并报告。

### DVP 为什么重要？

DVP 的价值在于降低本金风险。没有 DVP，就可能出现“钱付了但券没到”或“券交了但钱没到”。基金作为受托资产，最怕这种本金层面的安全问题。

所以这章记住一条主线：**交易追求成交，结算追求钱券安全，托管追求资产独立和合规复核。**

---

## 本课小结

投资交易与结算抓四句话：

1. 交易成交不等于结算完成。
2. 清算是算账，交收是钱券转移。
3. DVP 降低本金风险。
4. 托管人复核投资指令并参与资金划拨和交收监督。

一句话记忆：

> **前台负责买卖，后台负责让钱和券安全、准确、合规地到位。**
