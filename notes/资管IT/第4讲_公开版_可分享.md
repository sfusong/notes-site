## 一、本讲核心结论

1. **净值（NAV）是资管产品的"日结账单"**：每个交易日收市后，估值系统必须完成底层资产的重新定价，计算出当日NAV，这个数字决定了当日所有申赎交易的份额换算基准。
2. **摊余成本法和市值法是两种本质不同的资产定价逻辑**：摊余成本法让净值曲线平滑，市值法让净值曲线随市场波动；资管新规后理财子的估值方法选择受到明确约束。
3. **理财子产品普遍存在摊余成本法与市值法并用的情况**：同一产品内不同资产可能使用不同估值方法，大幅增加了估值系统的复杂度。
4. **日终清算不是一件事，而是三条并行的线**：资金清算、份额清算、费用清算。
5. **托管行复核净值是资管体系的重要制衡机制**：只有托管行确认后的NAV才能正式对外披露和用于申赎。
6. **影子定价是现金管理类产品的专用风险控制工具**：用于评估摊余成本法计算结果的公允性，偏离度超限有严格且不对称的监管处置要求。

## 三、核心概念与术语

### 3.1 净值（NAV）

```
NAV = （产品总资产 - 产品总负债）÷ 产品总份额

  产品总资产 = 所有底层资产的当日估值之和 + 现金及应收利息
  产品总负债 = 应付管理费 + 应付托管费 + 应付销售服务费 + 其他负债
  产品总份额 = TA系统中当日确认的总份额数
```

NAV计算错误的后果极其严重：NAV虚高则申购的投资者吃亏（买贵了）；NAV虚低则赎回的投资者吃亏（卖便宜了）。

### 3.2 摊余成本法（Amortized Cost Method）

**核心逻辑：** 债券等固定收益资产按买入成本入账，在持有期间将买入价与面值之间的溢价或折价均匀摊销到每一天，形成平滑净值曲线，不随市场利率变动而波动。

```
举例：以98元买入面值100元、持有到期1年的债券
  买入折价 = 2元，每日摊销 ≈ 0.00548元
  第1天账面价值 = 98.00548元（持续平滑上升直至到期日=100元）
  无论市场利率如何变化，账面价值按此曲线计算
```

**适用场景（资管新规约束后）：** 封闭式产品持有到期的固定收益资产；现金管理类理财产品（需配套影子定价）。

### 3.3 市值法（Mark-to-Market）

**核心逻辑：** 资产按当日市场价格（或公允价值）重新估值，每天的价格变动直接反映在NAV上。

**适用场景：** 股票、公开市场债券（非持有到期）、开放式净值型理财产品。

### 3.4 摊余成本法 vs 市值法对比

| 维度 | 摊余成本法 | 市值法 |
|------|-----------|--------|
| 计价基础 | 买入成本+每日摊销 | 当日市场公允价值 |
| 净值曲线 | 平滑，几乎单调递增 | 随市场波动，可能下跌 |
| 风险揭示 | 较弱，不反映市场风险 | 较强，实时反映市场变化 |
| 监管倾向 | 资管新规后受限 | 资管新规鼓励使用 |
| 典型适用产品 | 现金管理类、封闭式固收持有到期 | 公募基金、开放式净值型理财 |

> 资管新规限制的是摊余成本法的**滥用**（用摊余来掩盖可随时出售的资产的市场风险），而不是消灭摊余成本法本身。

### 3.5 中债估值

中央国债登记结算有限责任公司（中债登）发布的债券估值数据，是中国债券市场最权威的第三方估值来源。每个交易日收盘后发布，是市值法定价的主要依据。

### 3.6 影子定价（Shadow Pricing）

影子定价是监管要求采用摊余成本法的**现金管理类产品**必须配套的专用风险控制机制。

**定义：** 用市场公允价值重新计算一遍产品净值（影子NAV，记为NAVs），与摊余成本法计算的NAV（记为NAVa）进行比对，评估摊余成本法计算结果是否偏离市场公允价值过远。

| 概念 | 符号 | 定义 | 用途 |
|------|------|------|------|
| 摊余成本法NAV | NAVa | 按摊余成本法计算的每日净值 | 日常申赎定价、信息披露 |
| 影子定价NAV | NAVs | 按市场公允价值重新计算的净值 | 风险控制参照 |
| 偏离度 | — | (NAVs - NAVa) / NAVa | 衡量两种方法的差距，触发监管措施 |

> **重要区分**：影子定价不是"市值法"的另一种叫法，它是"为了监控摊余成本法是否偏离市场价值"而专门设计的风控工具，不用于日常申赎定价。

### 3.7 偏离度及监管处置要求

偏离度计算公式（来自银保监发〔2021〕20号）：

```
偏离度 = (NAVs - NAVa) / NAVa
```

**偏离度阈值与处置要求（正负不对称）：**

| 偏离方向 | 触发阈值 | 监管要求 | 时限 |
|----------|----------|----------|------|
| **正偏离** | ≥ 0.5% | 暂停接受认购 + 调整至0.5%以内 | 5个交易日 |
| **负偏离** | ≥ 0.25% | 调整至0.25%以内 | 5个交易日 |
| **负偏离** | ≥ 0.5% | 采取措施控制在0.5%以内 | 立即 |
| **负偏离** | 连续**2个**交易日 > 0.5% | 改用公允价值法调整账面价值，**或**暂停所有赎回并终止产品清算 | 立即强制 |

关键细节：负偏离0.25%就要开始处置，比正偏离（0.5%才触发）更严格；连续两日负偏离超0.5%，处置手段可能导致产品直接终止清算。

### 3.8 费用计提（Accrual）

按年化费率换算为每日应计提金额，在估值时从产品总资产中扣除，直接降低当日NAV。

> **重要区分**：费用计提只是账面记录（降低NAV），**不是每天实际划款**。实际的费用划付通常按月或按季度执行。这两个逻辑在系统设计上必须分开处理。

### 3.9 现金管理类产品的特殊规定

根据银保监发〔2021〕20号，与TA系统和估值系统直接相关的重要规定：

- **份额权益时间**：当日认购的份额自**下一个交易日**起享有分配权益；当日赎回的份额自下一个交易日起不享有分配权益
- **快速赎回上限**：单个投资者在单个销售渠道持有的单只产品单个自然日的赎回金额上限不高于**1万元**（即时到账），超过部分次日到账
- **规模约束**：同一理财公司采用摊余成本法的现管产品月末规模，合计不得超过其风险准备金月末余额的**200倍**

## 四、完整业务流程拆解

### 4.1 每日估值完整流程

```
【资管产品每日估值全流程】

  当日交易收盘（如下午3点）
        │
        ▼
  交易系统/前台系统
  · 锁定当日最终持仓数据，输出持仓快照
  · 推送给估值系统
        │
        ▼
  行情数据获取
  · 债券：从中债估值获取当日估值价格（收盘后约1-2小时内发布）
  · 股票：从交易所获取收盘价
  · 非标资产：使用约定的估值方法（成本法/独立评估）
        │
        ▼
  估值系统（核心计算）
  · 逐笔资产定价（市值法或摊余成本法）
  · 计算应计利息
  · 计提当日费用（管理费/托管费/销售服务费）
  · 计算当日NAV = （总资产 - 总负债）÷ 总份额
  · 现金管理类产品额外：计算影子定价NAV，计算偏离度，超限告警
  · 生成估值报告
        │
        ▼
  内部复核（自动校验+人工抽查）
        │
        ▼
  推送托管行独立复核
        │
  差异 ≤ 允许阈值          差异 > 允许阈值
        │                         │
        ▼                         ▼
  出具确认函                双方逐笔排查差异原因
  NAV正式生效               确认后重新出具确认函
        │
        ├─────────────────────┐
        ▼                     ▼
  推送TA系统              推送信息披露系统
  （用于份额清算）         （用于净值公告）
```

### 4.2 日终清算三条线

```
【日终清算三条线】

  ┌──────────────────────────────────────────────────┐
  │  资金清算（托管行主导）                              │
  │  · 申购资金归集确认                                 │
  │  · 赎回资金划付确认（托管户→投资者账户）             │
  │  · 投资交易资金交收（债券买卖的资金结算）             │
  └──────────────────────────────────────────────────┘
  ┌──────────────────────────────────────────────────┐
  │  份额清算（TA系统主导）                              │
  │  · 当日申购份额登记                                 │
  │  · 当日赎回份额注销                                 │
  │  · 产品总份额更新                                   │
  └──────────────────────────────────────────────────┘
  ┌──────────────────────────────────────────────────┐
  │  费用清算（估值系统计提 + 托管行定期划付）            │
  │  · 管理费/托管费/销售服务费/客户维护费 日计提         │
  │  · 重要：每日计提 ≠ 每日划款                         │
  │    计提是账面动作（降低NAV），实际划付按月/季度执行    │
  └──────────────────────────────────────────────────┘
```

### 4.3 托管行复核NAV的逻辑

```
管理人推送估值报告（持仓+计算过程+NAV结果）
        ↓
托管行用自己保存的持仓数据、行情数据、合同费率独立重新计算NAV
        ↓
比对管理人NAV vs 托管行独立计算NAV
        ↓
差异≤阈值 → 出具确认函，NAV正式生效
差异>阈值 → 双方逐笔排查（持仓/行情/费率/利息？），确认后重新出具确认函
```

**托管行复核的意义：** 防止管理人故意或失误抬高/压低净值；提供独立数据校验；是监管要求的合规程序，复核结果需留档备查。

### 6.2 非标资产估值的特殊处理

| 问题维度 | 标准债券 | 非标资产 |
|----------|---------|---------|
| 估值数据来源 | 中债登每日发布 | 独立估值机构定期提供，或按成本法 |
| 估值频率 | 每日 | 月度/季度，非每日更新 |
| 托管行复核难度 | 较低（有权威数据源） | 较高（缺乏公开市场价格参考） |

## 七、科技 / 产品经理视角应该关注什么

### 7.1 估值方法配置是"业务规则"，不是"系统参数"

估值系统的核心难点，不是计算逻辑本身，而是估值方法的配置必须与产品合同和监管规定完全一致。任何一个参数配错，都会导致NAV计算错误。

### 7.2 偏离度监控系统的设计要求

对于现金管理类产品，必须：
- 每日自动计算影子定价NAV和偏离度
- 设置分级告警（正偏离0.5%→暂停认购；负偏离0.25%→预警；负偏离0.5%→升级；连续2日超0.5%→紧急处置）
- 记录偏离度历史数据，备监管查阅
- 偏离度超限时有自动业务拦截（如自动暂停认购）

### 7.3 系统建设优先级

| 优先级 | 功能模块 | 理由 |
|--------|----------|------|
| P0 | 标准债券摊余成本法计算 | 理财子主要产品类型核心需求 |
| P0 | 中债估值数据接入 | 市值法唯一权威数据源 |
| P0 | 费用计提（管理费/托管费） | 每日必须执行 |
| P0 | 托管行数据推送与复核对接 | 合规强制要求 |
| P0 | 影子定价+偏离度监控 | 现金管理类产品监管强制要求 |
| P1 | 非标资产成本法处理 | 理财子特有需求 |
| P2 | 混合估值产品的灵活配置 | 复杂场景，可逐步建设 |

## 八、常见误区与易混点

### 误区1：摊余成本法比市值法"落后"
资管新规限制的是滥用，不是消灭摊余成本法。对于真正封闭持有到期的债券，摊余仍是允许且合理的方法。

### 误区2：NAV由管理人单方面决定
NAV必须经过托管行独立复核确认后才能正式生效。管理人单方面发布的NAV是不合规的。

### 误区3：费用计提就是每天把钱划走
费用计提只是账面记录（降低NAV），不是每天实际划款。实际划付通常按月或按季度执行。

### 误区4：影子定价就是"市值法NAV"
影子定价是为现金管理类产品专门设计的风控工具，用途是"给摊余成本法做参照校验"，而不是替代摊余成本法作为日常申赎的定价基础。

### 误区5：偏离度正负阈值对称
不对称。负偏离0.25%就要开始处置，比正偏离（0.5%才触发）更严格。这体现了监管对保护投资者利益（防止净值虚高坑了申购者）的优先考量。

---

## 九、案例分析

### 案例：某理财子"摊余转市值"改造引发的产品净值波动事件

**背景：** 某理财子旗下规模约30亿元的固收类理财产品，因开放频率从每季度改为每月，持有的部分债券不再满足摊余成本法适用条件，需要切换为市值法。

**切换过程发生了什么：**

切换前该债券按摊余计算的账面价值为100.3元，但切换当日中债估值只有98.5元（当日市场利率走高，债券价格下跌）。账面价值落差1.8元/张，该债券持仓6亿元，导致当日NAV下跌约0.36%，远超日常波动幅度，引发部分投资者恐慌，当月开放日赎回量明显上升。

**教训：**

| 问题 | 教训 |
|------|------|
| 切换时机选择不当 | 应在市场利率低位时切换，减少账面价值落差 |
| 未提前向投资者告知 | 估值方法重大变更应提前发布公告 |
| 系统切换未充分测试 | 需在测试环境完整演练后再上线 |
| 未提前与托管行沟通 | 重大参数变更应提前与托管行对齐 |

**核心价值：** 估值方法不是"技术配置"，是有重要业务影响的决策。产品经理必须理解估值方法变更对净值的影响，不能把它当成纯IT工作交给系统部门处理。

## 十、本讲小结

| 维度 | 核心要点 |
|------|----------|
| NAV定义 | （产品总资产 - 总负债）÷ 总份额，每日收盘后计算 |
| 摊余成本法 | 按买入成本均匀摊销，净值平滑，不随市场波动 |
| 市值法 | 按当日市场公允价值定价，净值随市场波动 |
| 影子定价 | 现管产品专用风控工具，用市场价格重算NAV以校验摊余的公允性 |
| 偏离度阈值 | 正偏离0.5%暂停认购；负偏离0.25%开始处置；不对称设计 |
| 日终清算三条线 | 资金清算 / 份额清算（TA主导） / 费用清算（计提≠划付） |
| 托管行复核 | 独立复核NAV是合规要求，复核确认后NAV才能正式生效 |
| 非标资产特殊性 | 无日频估值，成本法或定期独立估值，是估值体系最大复杂度来源 |

## 十二、课堂提问

**问题1：** 摊余成本法和市值法的本质区别是什么？资管新规对两种方法的使用是如何规定的？

> 💡 提示方向：从计价基础（历史成本vs公允价值）、净值曲线特征（平滑vs波动）、风险揭示程度三个维度展开。

---

**问题2：** 影子定价是什么？它和市值法有什么区别？偏离度超限了会怎样？（要求说出正负偏离的不同阈值）

> 💡 提示方向：影子定价是"给摊余成本法做参照校验的风控工具"，不是替代性的定价方法。偏离度阈值是不对称的，负偏离要求更严格。

---

**问题3：** 托管行为什么需要独立复核NAV？它独立复核的依据是什么？差异如何处理？

> 💡 提示方向：独立性是关键。差异处理需要双方逐笔排查，找出原因后才能确认最终NAV。

---

**问题4：** 日终清算三条线分别是什么？费用计提和实际划付有什么区别？

> 💡 提示方向：重点区分计提（每日账面动作，降低NAV）和划付（实际钱的流动，通常按月季度执行）。

---

**问题5：** 为什么说"非标资产是理财子估值体系最大的复杂度来源"？

> 💡 提示方向：从估值数据来源（无公开市场价格）、估值频率（非每日）、托管行复核难度四个维度展开。

## 十三、应该记住的关键句

**① `[业务理解]`**
> "NAV的计算不是一个简单的公式。每个交易日收盘后，估值系统要对每一笔底层资产重新定价——债券用中债估值，非标用约定的方法——扣除当日的管理费、托管费计提，才能算出当日净值。这个过程需要几个小时，这就是为什么TA系统的份额确认要在夜间批处理。"

**② `[业务理解]`**
> "摊余成本法和市值法的本质区别是：摊余按买入成本加每日均匀摊销，净值曲线很平滑；市值法按当日市场价格重新定价，净值随市场波动。资管新规限制的不是摊余本身，而是用摊余来掩盖可以随时出售的资产的市场风险。"

**③ `[业务理解]`**
> "影子定价不是市值法的另一个名字，它是专门为现金管理类产品设计的风险控制工具。产品用摊余成本法定价，但每天还要用市场价格重新算一遍影子净值，两者之差除以摊余NAV就是偏离度。偏离度的阈值是不对称的：负偏离0.25%就要开始处置，正偏离要到0.5%才暂停认购。"

**④ `[系统表达]`**
> "托管行独立复核NAV，是资管合规体系的核心制衡机制。管理人算的数不能自证清白，托管行用自己保存的持仓数据和行情数据独立算一遍，两边对上才能确认。NAV必须经托管行确认后才能对外披露，这是有法律效力的程序。"

**⑤ `[系统表达]`**
> "日终清算是三条并行的线：资金清算（钱的流向，托管行主导）、份额清算（份额增减，TA主导）、费用清算（管理费等计提加定期划付）。特别要注意：费用计提每天都在降低净值，但不是每天都把钱划走——计提是账面动作，实际划付通常按月季度执行，这两件事在系统设计上必须分开处理。"

**⑥ `[行业认知]`**
> "理财子估值比公募复杂，核心原因有三：第一，摊余和市值经常在同一个产品里混用；第二，非标资产没有每日市场价格，估值数据来自定期的独立评估机构；第三，现金管理类产品还需要配套影子定价和偏离度监控，这些对估值系统的设计要求非常高。"
