# 销售渠道与代销体系｜资管科技产品经理进阶课笔记（第6讲）

> **写在前面**：本笔记是系列课程《资管业务 × 科技系统 × 产品视角》第6讲的学习整理。前五讲建立了资管机构"内部"从产品设计、投资交易到TA系统、估值清算的完整认知，本讲从销售侧视角补齐全链条：理财产品是如何通过母行和代销渠道到达投资者手中的，以及渠道系统与TA系统之间的数据交互逻辑。
>
> **本讲新增内容**：在标准代销体系框架基础上，补充了"双录"监管要求（适当性管理的关键合规细节）、理财登在投资者侧的作用（防飞单机制）、以及批处理文件向实时API演进的行业趋势。

---

## 一、本讲核心结论

1. **银行理财子的销售渠道体系以母行为核心，兼有他行代销和直销**：母行渠道是最主要的销售来源，他行代销是规模扩张的重要手段，直销渠道是未来趋势。
2. **代销关系的本质是"委托销售协议"**：理财子委托代销机构销售产品，代销机构收取销售服务费和客户维护费，不承担产品亏损风险。
3. **投资者适当性管理（KYC+风险匹配+双录）是销售合规的核心**：匹配检查必须在申购前自动完成；R3及以上产品或特定人群还需满足"双录"监管要求。
4. **代销商与TA系统的数据交互是批处理模式**：T日晚文件→TA批处理→T+1早确认文件→代销更新持仓，这是当前模式；向实时API演进是行业趋势。
5. **多渠道管理是理财子科技系统建设的重要挑战**：不同代销商接口标准、文件格式不尽相同，是销售系统建设的核心复杂度来源。
6. **产品信息准确同步是销售合规的基础设施**：产品管理系统是唯一权威来源，信息不一致会直接引发合规问题和客诉。

## 二、行业背景与业务定位

### 2.1 理财子产品的销售渠道体系

```
【银行理财子销售渠道体系】

                    ┌─────────────────┐
                    │   银行理财子       │
                    │  （产品管理人）    │
                    └────────┬────────┘
                             │ 委托销售
          ┌──────────────────┼──────────────────┐
          │                  │                  │
          ▼                  ▼                  ▼
  ┌───────────────┐  ┌───────────────┐  ┌───────────────┐
  │   母行渠道     │  │  他行代销渠道  │  │   直销渠道     │
  │ 网点/APP/网银  │  │ 他行/券商/    │  │ 理财子官方APP/ │
  │               │  │ 第三方财富平台 │  │ 微信小程序    │
  └───────────────┘  └───────────────┘  └───────────────┘
                             ↓
                       投资者（零售/机构）
```

### 2.2 三种渠道模式的业务特点对比

| 维度 | 母行渠道 | 他行代销渠道 | 直销渠道 |
|------|----------|------------|---------|
| 重要性 | 最核心，规模最大 | 规模扩张的重要手段 | 占比较小，是未来方向 |
| 客户基础 | 母行存量客户 | 他行/平台客户 | 主动搜索产品的投资者 |
| 费用结构 | 销售服务费+客户维护费 | 同上，费率可能更高 | 无渠道费用 |
| 数据交互 | 相对标准（同集团内） | 接口多样，需逐家对接 | 理财子自建系统 |
| 客户数据归属 | 在母行手中 | 在代销商手中 | **在理财子手中** |

### 2.3 理财子渠道体系的历史背景

银行理财子成立前，银行理财产品一直通过银行自身渠道销售。理财子成立后，产品管理职能从银行独立出来，但销售渠道依然高度依赖母行。这与公募基金有本质差异——公募基金的产品与销售渠道天然分离，而理财子的产品与母行渠道天然捆绑。

## 三、核心概念与术语

### 3.1 代销协议（Distribution Agreement）

| 条款类型 | 主要内容 |
|----------|---------|
| 产品范围 | 哪些产品可以通过该渠道销售 |
| 销售规则 | 投资者适当性要求、销售限额、宣传材料规范 |
| 费用条款 | 销售服务费率、客户维护费率、费用计算和支付方式 |
| 数据交互 | 申赎文件的格式、传输方式、时间要求 |
| 合规要求 | 代销机构的合规义务、信息披露要求、双录要求 |

### 3.2 销售服务费与客户维护费

从产品资产中按日计提，定期划付给代销商，不向投资者单独收取申购/赎回手续费（理财子与公募基金费率结构的主要差异之一）：

| 费用类型 | 计费基础 | 用途 |
|----------|---------|------|
| 销售服务费 | 产品规模 × 年化费率 | 覆盖渠道销售成本，包括客户经理销售提成等 |
| 客户维护费 | 产品规模 × 年化费率 | 支持代销商对投资者的持续服务和维护 |

### 3.3 投资者适当性管理（Investor Suitability）

**产品风险等级（五级分类）：**

| 风险等级 | 描述 | 对应投资者 |
|----------|------|----------|
| R1（谨慎型） | 本金安全，收益稳定 | 保守型及以上 |
| R2（稳健型） | 本金较安全，收益较稳定 | 稳健型及以上 |
| R3（平衡型） | 本金有一定风险，收益有波动 | 平衡型及以上 |
| R4（进取型） | 本金风险较高，收益波动大 | 进取型及以上 |
| R5（激进型） | 本金风险高，可能损失全部本金 | 激进型 |

**匹配规则：** 产品风险等级必须 ≤ 投资者风险评级（低风险产品可卖给高风险承受能力投资者，反之不行）。

**风险评级有效期：** 通常1-3年，过期需重新评估；系统须自动检查。

### 3.4 "双录"监管要求

"双录"是指对销售特定理财产品时进行**录音录像**，以留存合规证据。

**触发条件：**
- 销售风险等级R3及以上的产品
- 向65岁（部分机构为60岁）及以上老年人销售任何风险等级的产品

**对销售系统设计的影响：**

| 设计要点 | 说明 |
|----------|------|
| 触发判断 | 系统自动判断是否需要双录，不依赖人工判断 |
| 双录内容 | 风险提示朗读确认、投资者身份核验、产品关键信息确认 |
| 数据存储 | 录音录像文件须与申购记录关联存储，保存期限通常≥5年 |
| 失败处理 | **双录失败（投资者拒绝录制）则不得完成申购，系统须阻断流程** |
| 线上实现 | 线上APP需设计视频录制流程，投资者须打开摄像头和麦克风 |

### 3.5 理财登在投资者侧的作用

**中国理财网（www.chinawealth.com.cn）** 是理财登旗下的官方信息披露平台，投资者可以输入产品名称或产品登记编码（Z开头），查询该产品是否是正规理财子发行的真实产品，以及产品的基本信息、历史净值等。

这是防止投资者被假冒理财产品诈骗（"飞单"）的重要机制，也是理财子产品合规销售的配套保障。

> **注意**：理财登是监管层面的信息披露平台，代销商的销售系统需要理财子主动推送产品信息，并不会自动读取理财登的数据。两者是并行的信息流，而非主从关系。

### 3.6 申赎文件（Transaction File）

代销商与TA系统之间批量传输交易数据的文件，典型内容：

| 数据类型 | 字段示例 |
|----------|---------|
| 账户类申请 | 账户开立/变更、风险评级更新 |
| 申购申请 | 投资者账号、产品代码、申购金额、申购时间 |
| 赎回申请 | 投资者账号、产品代码、赎回份额/金额 |

**文件传输规范**（在代销协议中约定）：文件格式（CSV/固定长度/XML等）、传输方式（SFTP/专线/API等）、截止时间（如T日晚8点前）。

### 3.7 批处理文件 vs 实时API：行业演进趋势

**当前现状（批处理文件模式）：**
代销商T日收盘后汇总申赎请求，生成文件发TA系统；TA系统夜间批处理，T+1早完成份额确认。优点：对账清晰，历史沿袭成本低。缺点：时效性差，节点延迟影响整批次。

**演进方向（实时API模式）：**
代销商实时推送投资者下单申请，TA系统实时响应状态，用户可立即看到受理状态。优点：用户体验更好，异常处理更及时。挑战：TA系统需要支持高并发实时处理，代销商系统改造成本高，现有批处理对账逻辑需重新设计。

## 四、完整业务流程拆解

### 4.1 代销合作建立的完整流程

```
理财子商务洽谈，确定产品范围和费率
        ↓
签署代销协议（含数据交互规范+双录要求+费用条款）
        ↓
系统对接（双方IT联调申赎文件格式和传输通道）
        ↓
产品信息同步（理财子推送产品信息，代销商内部审核）
        ↓
代销商内部上架
（配置产品展示信息/风险等级/适当性规则/双录触发规则）
        ↓
正式开始销售
```

### 4.2 投资者申购全流程（代销渠道视角，含双录）

```
【投资者申购全流程（代销渠道视角）】

T日（交易日）：

  投资者登录代销渠道APP/网银
        ↓
  代销销售系统展示产品列表和详情
        ↓
  适当性检查（申购前自动完成）
  · 读取投资者风险评级（含有效期检查）
  · 读取产品风险等级
  · 检查：投资者风险评级 ≥ 产品风险等级？
    ✓ 通过：继续
    ✗ 不通过：拒绝申购，提示原因
        ↓
  双录检查（如需）
  · 判断是否触发双录：产品风险等级≥R3，或投资者年龄≥65岁？
    → 是：引导投资者完成录音录像
    → 否：跳过
  · 双录失败（投资者拒绝）→ 阻断申购，不得完成
        ↓
  投资者确认申购（填写金额+阅读风险提示+提交）
        ↓
  代销系统冻结申购资金，生成申购订单
  如有双录：录音录像文件与订单关联存储

T日收盘后：

  代销系统汇总当日所有交易申请
  → 按约定格式生成申赎文件
  → 发送给理财子TA系统（截止如晚8点）

  TA系统批处理（第3讲已详细拆解）
  → 完成份额清算 → 生成确认文件发回代销系统

T+1日：

  代销系统读取TA确认文件
  → 更新投资者持仓显示
  → 按确认金额将申购款划转至管理人过渡户
  → 投资者在APP上看到持仓
```

### 4.3 产品信息同步流程

```
产品管理系统（理财子内部，唯一权威来源）
  产品信息变更时自动推送
        ↓
  ┌──────────────────────────────────┐
  │  母行销售系统 / 他行代销系统 / 直销平台 / 理财登  │
  └──────────────────────────────────┘
  更新产品货架信息、展示页面、开放日历配置、
  适当性匹配配置、双录触发规则
        ↓
  投资者端看到更新后的产品信息
```

### 4.4 费用结算流程

```
估值系统每日计提销售服务费+客户维护费（降低产品NAV）
        ↓（按月/季度）
托管行将计提费用划付至代销商账户
        ↓
代销商内部分配（客户经理提成/投资者维护等）
```

### 6.2 多代销渠道接口的复杂度

```
理财子TA系统
  母行（同集团）：内部直连，格式标准
  代销商A（他行）：CSV文件，SFTP传输，字段集合X
  代销商B（他行）：固定长度文本，专线传输，字段集合Y
  代销商C（第三方平台）：XML文件，API传输，字段集合Z

→ 理财子需要维护N套不同的数据交互规范
→ 任何一套接口出问题，当日该代销商所有申赎都受影响
```

## 七、科技 / 产品经理视角应该关注什么

### 7.1 代销商文件接口管理是高频运营风险点

| 风险场景 | 业务影响 | 应对设计 |
|----------|---------|---------|
| 代销商文件格式变更未通知 | TA系统解析失败，当日批处理中断 | 建立接口变更通知机制，变更前联调测试 |
| 文件传输超时 | TA无法按时完成批处理 | 设置超时告警，自动通知运营人员和代销商 |
| 文件内字段值非法 | TA校验不通过，该代销商所有申请挂起 | 文件接收后先做格式校验，立即反馈代销商 |
| 代销商重复发送同一文件 | 可能导致重复处理 | TA系统做幂等处理，同一批次号只处理一次 |

### 7.2 适当性管理+双录的系统设计要点

| 设计要点 | 说明 |
|----------|------|
| 风险评级有效期 | 系统自动检查，过期则强制重新评估，不得绕过 |
| 双录触发判断 | 系统自动判断，不依赖人工；双录失败须阻断申购 |
| 双录文件存储 | 与申购记录关联存储，按监管要求保存≥5年 |
| 特殊情形留痕 | 投资者主动要求买超出等级产品时，系统记录主动确认 |

### 7.3 产品信息同步的"单一权威来源"原则

任何渠道系统都不应该独立维护产品信息。产品管理系统是唯一权威来源，变更自动推送，不依赖人工通知。

**信息不一致的典型风险：**

| 场景 | 后果 |
|------|------|
| 开放日调整后未及时通知代销商 | 代销商APP仍显示旧开放日，投资者申购被拒→客诉 |
| 产品风险等级变更后未同步 | 适当性匹配按原等级执行，可能引发合规问题 |
| 产品提前终止但代销商未下架 | 投资者仍能提交申购，申购被拒→客诉 |

## 八、常见误区与易混点

### 误区1：理财子直接管理投资者账户
理财子通过TA系统管理"产品层面的份额账户"，不是"投资者在代销机构的资金账户"。

### 误区2：代销商文件里的金额就是最终申购金额
文件里是"申购申请金额"，最终确认金额可能因产品规模上限、大额申购保护等原因而不同，T+1早才能知道最终结果。

### 误区3：投资者风险评级一次评估终身有效
风险评级有有效期（通常1-3年），系统必须自动检查。

### 误区4：双录只是"录个视频"这么简单
双录有明确的监管触发条件，有具体的内容要求，有严格的存储要求（关联申购记录、保存5年以上），双录失败不得完成申购，系统必须阻断。

### 误区5：产品信息在理财登登记了，代销商系统就自动更新
理财登是监管信息披露平台，不等于代销商销售系统的数据来源。两者是并行的信息流，需要理财子分别维护。

---

## 九、案例分析

### 案例：某理财子产品开放日调整引发的多渠道信息不一致问题

**背景：** 某理财子旗下月度开放型固收类理财产品，需要将每月申购开放日从1日调整为5日（次月生效）。该产品通过母行渠道和两家他行代销渠道销售，共约3.2万名持有人。

**事件经过：**

理财子在理财登更新了开放日历，并向母行发出了变更通知，但未及时向两家他行代销商发出通知。变更生效后首个原开放日（每月1日）：母行渠道已正确更新系统；两家他行代销商系统未更新，仍显示"今日开放申购"。大量投资者提交申购申请，代销商在T日晚发送了申赎文件，TA系统校验发现该产品当日不在开放状态，所有申请被拒绝。T+1日代销商读取确认文件后，大量投资者收到申购失败通知，客诉电话涌入。

**改进措施：**

| 问题根源 | 改进措施 |
|----------|---------|
| 产品信息变更通知流程不完善 | 建立标准化变更通知清单，必须获得代销商系统更新的确认回执 |
| 产品管理系统没有自动推送机制 | 升级为关键信息变更自动推送至所有代销商系统 |
| 缺乏渠道信息一致性监控 | 建立每日自动比对机制，发现不一致立即告警 |

**核心价值：** 产品信息变更的影响范围不只是理财子内部系统，还包括所有代销渠道。"人工通知"是不可靠的同步机制，系统层面的自动推送才是正确解决方案。

## 十、本讲小结

| 维度 | 核心要点 |
|------|----------|
| 销售渠道体系 | 母行渠道（核心）+ 他行代销（扩张）+ 直销（趋势） |
| 代销关系本质 | 委托销售协议，代销商不承担产品亏损风险，收取销售服务费+客户维护费 |
| 适当性管理 | KYC+风险匹配+双录（R3及以上或特定人群），申购前由销售系统自动执行 |
| 双录要求 | R3及以上产品或65岁以上老年人必须双录，失败则阻断申购，文件存储≥5年 |
| 理财登投资者侧 | 投资者可通过中国理财网输入Z开头编码核实产品真实性（防飞单） |
| 代销商与TA系统 | 批处理模式：T日晚文件→TA批处理→T+1早确认文件→代销更新持仓 |
| 行业演进趋势 | 批处理文件→实时API，用户体验更好，但TA系统架构需要重构 |
| 产品信息同步 | 产品管理系统是唯一权威来源，变更自动推送，不依赖人工通知 |

## 十二、课堂提问

**问题1：** 理财子的销售渠道体系和公募基金有什么本质区别？母行渠道和他行代销渠道在数据交互上有什么差异？

> 💡 提示方向：公募产品与渠道天然分离；理财子产品与母行渠道天然捆绑。母行通常在同集团内，接口标准更统一；他行代销接口规范各异，管理复杂度更高。

---

**问题2：** "双录"监管要求的触发条件是什么？它对销售系统设计有哪些具体影响？双录失败了怎么办？

> 💡 提示方向：R3及以上产品或65岁以上老年人触发双录。系统自动判断触发条件，双录失败须阻断申购，双录文件与申购记录关联存储≥5年。

---

**问题3：** 理财登在投资者侧有什么具体作用？它和代销商销售系统的产品信息是什么关系？

> 💡 提示方向：中国理财网允许投资者用Z开头编码核实产品真实性，是防飞单的重要机制。理财登和代销商销售系统是并行的信息流，需要理财子分别维护。

---

**问题4：** 代销商与TA系统的数据交互为什么采用批处理文件模式？这种模式相比实时API有什么优缺点？行业未来会往哪个方向走？

> 💡 提示方向：批处理模式对账清晰，历史沿袭成本低，但时效性差。实时API用户体验更好，但TA系统架构需要从批处理变成实时处理，改造成本不小。

---

**问题5：** 为什么说"产品管理系统向代销渠道的信息同步是销售合规的基础设施"？产品信息在不同渠道出现不一致，会带来哪些具体风险？

> 💡 提示方向：开放日不一致→申购被拒→客诉；风险等级不一致→适当性合规风险；产品终止后未下架→无效申购→客诉。三类风险都有真实发生的案例。

## 十三、应该记住的关键句

**① `[业务理解]`**
> "理财子的销售渠道体系和公募基金不一样。公募基金的产品和销售渠道天然分离；理财子的产品和母行渠道天然捆绑。母行是最主要的销售来源，他行代销是扩大规模的手段。但每家代销商的接口规范不一样，TA系统要维护N套不同的数据交互规范，这是多渠道管理最主要的挑战。"

**② `[业务理解]`**
> "投资者适当性管理有三个层次：KYC产生风险评级、申购前做风险匹配检查、R3及以上产品或特定人群要做双录。双录不是可选项——触发条件满足但双录失败，系统必须阻断申购流程，不得绕过。双录文件要与申购记录关联存储，通常是五年以上。"

**③ `[系统表达]`**
> "代销商和TA系统之间是批处理文件模式：代销商在T日收盘后汇总当天所有申赎请求，生成文件发给TA系统；TA系统批处理完成份额清算后，把确认文件发回代销商；代销商T+1早上读取确认文件，才能更新投资者持仓。行业在往实时API方向演进，用户体验更好，但TA系统的架构需要重构，改造成本不小。"

**④ `[业务理解]`**
> "理财登旗下有一个叫中国理财网的官方平台，投资者可以输入产品的Z开头编码，查询这个产品是不是正规理财子发行的真实产品。这是防飞单的重要机制——解决了投资者无法自己核实产品真实性的问题。"

**⑤ `[系统表达]`**
> "产品管理系统是所有销售渠道产品信息的唯一权威来源，任何渠道系统都不应该独立维护产品信息。产品信息变更必须触发自动推送，不依赖人工通知——一旦靠人工通知，就容易有代销商漏掉，导致投资者在错误日期申购被拒或者买到风险等级匹配不上的产品。"
