5. **建立一套可持续更新的行业认知框架**，为后续9讲奠定认知基础。

## 二、本讲核心结论

1. **中国资管行业经历了从"野蛮生长"到"强监管规范化"的结构性转变**，2018年资管新规是最重要的分水岭，理解新规前后的变化，是理解当前行业格局的前提。
2. **资管行业本质是"受托理财"**：机构替客户管钱、选资产、控风险，并从中收取管理费，核心是信托责任（Fiduciary Duty）。
3. **资管机构的分类逻辑有两条主线**：一是按照"谁发行产品"（公募 vs 私募）；二是按照"谁监管"（证监会 vs 银保监/金融监管总局）。
4. **每一类资管机构都有其特定的客户群体、产品形态和监管约束**，这决定了它们对科技系统的需求差异极大。
5. **资管科技的价值不在于"炫技"，而在于支撑合规、提升效率、管控风险**，科技人员必须理解业务才能做出有价值的系统。
6. **银行理财子是当前中国资管行业最具成长性的细分赛道之一**，其科技建设尚处于起步阶段，存在大量产品经理和系统建设机会。

## 四、行业背景与业务定位

### 4.1 什么是资产管理

资产管理（Asset Management），简称"资管"，是指金融机构接受投资者的委托，代其管理资产、进行投资运作，并收取相应报酬的金融服务。

这个定义里有几个关键词：
- **委托**：投资者把钱交给机构，形成"委托-代理"关系
- **代管理**：机构代替投资者做投资决策
- **收费**：机构收取管理费、托管费、业绩报酬等

与银行存款（刚性兑付、本金保障）不同，资管产品通常是**净值化产品**，意味着客户需要自担投资损益，机构不承诺保本。这一点在2018年资管新规之后被强制推行，是理解中国资管行业现状的重要背景。

### 4.2 资管 vs 财富管理

这两个概念经常被混用，但有本质区别：

| 维度 | 资产管理（Asset Management） | 财富管理（Wealth Management） |
|------|------------------------------|-------------------------------|
| 服务对象 | 机构，或批量客户 | 高净值个人客户 |
| 核心工作 | 管理投资组合，选股选债 | 综合金融规划，产品配置 |
| 收入来源 | 管理费、业绩报酬 | 顾问费、销售佣金 |
| 典型机构 | 公募基金、理财子 | 私人银行、独立理财顾问 |

简而言之：**资管是"管钱"，财富管理是"配钱"**。前者的核心能力是投资管理，后者的核心能力是客户关系和配置建议。

### 4.3 中国资管行业的发展阶段

| 阶段 | 时间 | 主要特征 |
|------|------|----------|
| 起步期 | 1998年前后 | 公募基金设立，资管概念引入 |
| 快速扩张期 | 2005–2017年 | 各类机构涌入，规模野蛮增长，刚性兑付盛行 |
| 强监管整顿期 | 2018–2021年 | 资管新规出台，打破刚兑，净值化改造 |
| 规范化发展期 | 2021年至今 | 理财子公司设立，行业格局重塑，科技化建设提速 |

**资管新规（2018年）的核心影响：**

2018年4月，中国人民银行等多部委联合发布《关于规范金融机构资产管理业务的指导意见》（即"资管新规"），其核心要求包括：
- **打破刚性兑付**：资管产品不得承诺保本保收益
- **净值化管理**：资管产品必须以净值形式披露
- **禁止多层嵌套**：限制资金池运作，穿透管理
- **合格投资者要求**：明确私募产品的投资者门槛

这一规定直接导致银行理财业务从银行表内剥离，推动了**银行理财子公司**的大规模设立。宁银理财正是在这个背景下成立的。

### 4.4 中国资管市场规模

根据中国资管行业的公开数据，截至2023年底，中国资管市场总规模约在**130万亿元人民币**左右。

| 细分赛道 | 约占规模 | 主要机构举例 |
|----------|----------|--------------|
| 公募基金 | 约27万亿元 | 易方达、华夏、嘉实、广发 |
| 银行理财（含理财子） | 约26–28万亿元 | 招银理财、兴银理财、宁银理财 |
| 保险资管 | 约25万亿元+ | 平安资管、国寿资管 |
| 券商资管 | 约7–9万亿元 | 中信证券资管、广发资管 |
| 私募基金 | 约20万亿元 | 景林、高毅、九坤（量化） |
| 信托 | 约20万亿元 | 中信信托、华润信托 |

> 注意：上述数字会随市场波动和监管口径不同而变化，在面试中只需了解量级和相对关系，不必死记硬背具体数字。

## 五、核心概念与术语

### 5.1 公募 vs 私募

| 维度 | 公募（Public Fund） | 私募（Private Fund） |
|------|---------------------|----------------------|
| 募集方式 | 面向不特定社会公众 | 面向特定合格投资者 |
| 投资门槛 | 1元起（货基）/1万元起 | 通常100万元起 |
| 信息披露 | 严格，须定期公告净值 | 相对宽松 |
| 监管机构 | 证监会 | 证监会（基金业协会登记） |
| 典型产品 | 公募股票基金、债基、货基 | 私募股权、对冲基金 |

### 5.2 理财子公司

银行理财子公司（简称"理财子"）是指由商业银行全资或控股设立的从事理财业务的非银行金融机构。

设立理财子的目的，是将原先银行表内理财业务剥离出来，实现与银行自营业务的风险隔离，并按照资管新规进行规范化运作。

理财子的特点：
- **母行资源优势**：依托银行的客户基础和渠道网络
- **固收主导**：产品以固定收益类为主，权益类占比低
- **净值化改造**：必须按净值管理，不能刚性兑付
- **监管机构**：主要由国家金融监督管理总局（原银保监会）监管

### 5.3 托管行（Custodian Bank）

托管行是指独立保管资管产品资产（资金和证券），并对管理人的投资行为进行监督的金融机构。

托管行的核心职责：
- 保管资产（资金划转、证券保管）
- 复核资产净值（与管理人进行对账）
- 监督投资合规（检查是否违反投资限制）
- 出具资金划付指令的复核

托管行在资管系统建设中是一个关键的外部角色，TA系统与托管行系统之间的数据对接，是你在宁银理财工作中实际接触到的重要内容。

### 5.4 TA系统（Transfer Agent System）

TA，即Transfer Agent（登记过户代理人），是负责管理投资者账户、处理申购/赎回交易、计算份额、分配收益等工作的核心业务系统。

在基金行业，TA系统是最核心的后台系统之一，其重要性类似于银行的核心账务系统。具体内容将在第3讲深入讲解，本讲先建立概念。

### 5.5 净值（NAV，Net Asset Value）

基金净值是指基金单位资产的价值，计算公式为：

```
基金单位净值 = （基金总资产 - 基金总负债）/ 基金份额总数
```

净值是资管产品定价的核心，所有申购赎回交易都以净值为基础进行结算。每日净值的计算和披露，是资管机构运营工作中的重要日常任务。

### 5.6 份额（Share/Unit）

基金份额是投资者持有基金的计量单位。投资者用金额申购，基金公司根据净值换算成份额。

例如：投资者申购10,000元，当日净值为1.05元，则获得 10,000 / 1.05 ≈ 9,523.81 份。

## 九、站在科技 / 产品 / 项目经理视角应该关注什么

### 9.1 监管是业务需求的隐性来源

资管机构的很多系统需求，本质上来自监管要求。例如，净值每日披露的要求，决定了估值系统必须在每个交易日收盘后完成计算；资管新规对多层嵌套的限制，要求系统必须能追踪产品底层资产的穿透数据。因此，读懂监管文件，是资管产品经理的基本功。

### 9.2 业务流程的标准化程度参差不齐

公募基金行业经过20多年发展，业务流程已经高度标准化，系统供应商的解决方案相对成熟。但银行理财子成立时间短（大部分成立于2019–2020年），很多业务流程仍处于建立和优化阶段，科技建设需求旺盛，这也是为什么理财子是当前最好的科技PM职业赛道之一。

### 9.3 外部对接是系统建设的主要复杂度来源

资管系统不是孤立运行的，它需要与托管行、中国结算、代销渠道、监管报送平台等多个外部系统进行数据交换。这些接口往往规范不一、响应不稳定，是系统建设中最容易出问题的环节。

### 9.4 产品经理的核心价值定位

| 能力维度 | 具体表现 |
|----------|----------|
| 业务翻译能力 | 能把业务需求翻译成系统语言，反之亦然 |
| 流程设计能力 | 能梳理端到端业务流程，识别系统断点 |
| 外部协作能力 | 能与供应商、托管行、监管机构沟通 |
| 项目管理能力 | 能推进多方协同的系统建设项目 |
| 合规意识 | 理解监管边界，不做踩红线的需求设计 |

## 十一、常见误区与易混点

### 误区1：把"资管"和"理财"混为一谈

很多人认为"资管=银行理财"，实际上银行理财只是资管行业的一个细分品类。资管行业更大，包括公募基金、私募基金、保险资管、券商资管等多个板块。银行理财子，是资管行业中近年来快速崛起的一个细分赛道。

### 误区2：认为"净值化"就是"有风险"

净值化管理确实意味着产品没有保本承诺，但这不代表所有净值型产品都是高风险的。货币市场基金、低波动的固收类理财产品，同样是净值化的，但波动极小。净值化是管理形式，不直接等同于风险等级。

### 误区3：把"托管行"和"存管行"混淆

- **托管行（Custodian Bank）**：负责管理资管产品的资产，进行独立监督
- **存管行（Depository Bank）**：负责保管证券公司客户的资金和证券（面向证券公司客户，而非基金产品）

在资管语境下，通常说的是托管行，不是存管行。

### 误区4：认为"TA系统"只做"份额登记"

TA系统的职责远不止份额登记，还包括：申赎交易处理、收益分配计算、分红业务处理、历史交易查询、对账管理、监管报送等。TA系统是资管后台系统中功能最集中、数据最敏感的核心系统之一。

### 误区5：认为"科技部门"不需要懂业务

在资管机构，科技部门（尤其是产品经理和业务分析岗）如果不懂业务，会直接导致系统设计错误、需求理解偏差、项目延期等问题。资管科技PM必须具备相当程度的业务理解才能有效工作。

## 十二、案例分析

### 案例：某理财子新设背景下的系统建设规划

**背景：**

某城商行于2020年成立了全资理财子公司（与宁银理财成立背景类似），初期团队规模约50人，管理规模约800亿元。由于公司成立时间短，大量业务流程依赖手工操作和Excel，主要痛点包括：
- 产品信息散落在各部门邮件和Excel表格中，无法统一查询
- 申赎数据需要人工在托管行系统和内部Excel之间手动核对
- 监管报送数据汇总费时费力，容易出错
- 产品上线审批流程依赖线下纸质签署，效率低

**科技建设方案：**

```
第一阶段（0–12个月）：
  优先级1：TA系统上线
    → 解决申赎处理和份额登记的自动化问题
    → 与托管行建立数据接口
    
  优先级2：产品管理系统建设
    → 统一产品信息管理
    → 上线审批流程电子化

第二阶段（12–24个月）：
  优先级3：估值清算系统优化
    → 减少人工干预
    → 提升净值计算准确性

  优先级4：监管报送系统对接
    → 自动化生成报送数据
    → 降低合规风险
```

**产品经理的核心挑战：**

1. **需求拉齐**：业务部门、运营部门、合规部门对系统的期望不一致，需要产品经理作为桥梁协调
2. **供应商选型**：市场上TA系统供应商有恒生电子、中科软、金证股份等，每家的功能侧重和实施能力不同，需要进行全面的尽调和评估
3. **数据迁移**：历史业务数据（存在Excel中）需要清洗、标准化后才能导入新系统，数据质量问题往往是项目延期的主要原因
4. **托管行联调**：与托管行系统的接口联调涉及两个机构的IT团队协同，时间周期难以控制

这个案例与在宁银理财的实际工作场景高度吻合。
