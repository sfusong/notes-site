## 二、本讲核心结论

1. **净值（NAV）是资管产品的"日结账单"**：每个交易日收市后，估值系统必须完成底层资产的重新定价，计算出当日NAV，这个数字决定了当日所有申赎交易的份额换算基准。
2. **摊余成本法和市值法是两种本质不同的资产定价逻辑**：摊余成本法让净值曲线平滑，市值法让净值曲线随市场波动；资管新规后理财子产品的估值方法选择受到明确约束。
3. **理财子产品普遍存在摊余成本法与市值法并用的情况**：同一产品内不同资产可能使用不同估值方法，大幅增加了估值系统的复杂度。
4. **日终清算不是一件事，而是三条并行的线**：资金清算、份额清算、费用清算，三条线结果汇总后才能完成完整的日终处理。
5. **托管行复核净值是资管体系的重要制衡机制**：只有托管行确认后的NAV才能正式用于申赎和对外披露。
6. **影子定价是现金管理类产品的专用风险控制工具**：用于评估摊余成本法计算结果的公允性，偏离度超限有严格且不对称的监管处置要求。

## 三、为什么这一讲重要（结合职业目标）

**第一层：这是理解TA系统工作的"上游逻辑"**

你在TA系统工作时，知道TA跑批需要等估值系统的行情数据。本讲把这个上游逻辑补齐：估值系统为什么需要这么长时间、为什么会延迟、延迟了会影响什么。

产品经理在设计产品时必须确定估值方法，这直接影响净值波动特征，进而影响产品对投资者的吸引力。

**第三层：影子定价和偏离度是现管产品的特有监管要求**

能清楚说出影子定价的机制和偏离度的处置阈值，是面试中涉及现金管理类产品时的显著加分点。

## 五、核心概念与术语

### 5.1 净值（NAV）

```
NAV = （产品总资产 - 产品总负债）÷ 产品总份额

  产品总资产 = 所有底层资产的当日估值之和 + 现金及应收利息
  产品总负债 = 应付管理费 + 应付托管费 + 应付销售服务费 + 其他负债
  产品总份额 = TA系统中当日确认的总份额数
```

### 5.2 摊余成本法（Amortized Cost Method）

**核心逻辑：** 债券等固定收益资产按**买入成本**入账，在持有期间将买入价与面值之间的溢价或折价**均匀摊销**到每一天，形成平滑净值曲线，不随市场利率变动而波动。

```
举例：以98元买入面值100元、持有到期1年的债券
  买入折价 = 2元，每日摊销 ≈ 0.00548元
  第1天账面价值 = 98.00548元
  第2天账面价值 = 98.01096元
  ...持续平滑上升，到期日 = 100元
  无论市场利率如何变化，账面价值按此曲线计算
```

**适用场景（资管新规约束后）：**
- 封闭式产品持有到期的固定收益资产
- 现金管理类理财产品（需配套影子定价）

### 5.3 市值法（Mark-to-Market）

**核心逻辑：** 资产按**当日市场价格**（或公允价值）重新估值，每天的价格变动直接反映在NAV上。

```
举例：产品持有某债券
  昨日中债估值 = 99.5元
  今日市场利率上升，债券价格下跌
  今日中债估值 = 98.8元
  今日该债券对净值贡献比昨日少了0.7元/张
  这个变动直接体现在当日NAV下跌中
```

**适用场景：** 股票、公开市场债券（非持有到期）、开放式净值型理财产品。

### 5.4 摊余成本法 vs 市值法对比

| 维度 | 摊余成本法 | 市值法 |
|------|-----------|--------|
| 计价基础 | 买入成本+每日摊销 | 当日市场公允价值 |
| 净值曲线 | 平滑，几乎单调递增 | 随市场波动，可能下跌 |
| 风险揭示 | 较弱，不反映市场风险 | 较强，实时反映市场变化 |
| 监管倾向 | 资管新规后受限 | 资管新规鼓励使用 |
| 典型适用产品 | 现金管理类、封闭式固收持有到期 | 公募基金、开放式净值型理财 |

### 5.5 中债估值

中央国债登记结算有限责任公司（中债登）发布的债券估值数据，是中国债券市场最权威的第三方估值来源。每个交易日收盘后发布，覆盖银行间和交易所市场绝大多数债券，是市值法定价的主要依据。

### 5.6 影子定价（Shadow Pricing）

影子定价是监管要求采用摊余成本法的**现金管理类产品**必须配套的专用风险控制机制。

**定义：** 用市场公允价值重新计算一遍产品净值（影子NAV），与摊余成本法计算的NAV进行比对，评估摊余成本法计算结果是否偏离市场公允价值过远。

| 概念 | 符号 | 定义 | 用途 |
|------|------|------|------|
| 摊余成本法NAV | NAVa | 按摊余成本法计算的每日净值 | 日常申赎定价、信息披露 |
| 影子定价NAV | NAVs | 按市场公允价值重新计算的净值 | 风险控制参照 |
| 偏离度 | — | (NAVs - NAVa) / NAVa | 衡量两种方法的差距，触发监管措施 |

> **重要区分**：影子定价不是"市值法"的另一种叫法，它是"为了监控摊余成本法是否偏离市场价值"而专门设计的风控工具，不用于日常申赎定价。

### 5.7 偏离度及监管处置要求

偏离度计算公式（来自银保监发〔2021〕20号）：

```
偏离度 = (NAVs - NAVa) / NAVa
```

**偏离度阈值与处置要求（正负不对称）：**

| 偏离方向 | 触发阈值 | 监管要求 | 时限 |
|----------|----------|----------|------|
| **正偏离** | ≥ 0.5% | 暂停接受认购 + 调整至0.5%以内 | 5个交易日 |
| **负偏离** | ≥ 0.25% | 调整至0.25%以内 | 5个交易日 |
| **负偏离** | ≥ 0.5% | 采取措施控制在0.5%以内 | 立即 |
| **负偏离** | 连续**2个**交易日 > 0.5% | 改用公允价值法调整账面价值，**或**暂停所有赎回并终止产品清算 | 立即强制 |

关键细节：负偏离0.25%就要开始处置，比正偏离更严格；连续两日负偏离超0.5%，处置手段可能导致产品直接终止清算。

### 5.8 费用计提（Accrual）

按年化费率换算为每日应计提金额，在估值时从产品总资产中扣除，直接降低当日NAV。

```
日费用计提额 = 产品资产规模 × 年化费率 ÷ 365

例：产品规模10亿元，管理费年化0.5%
  日计提管理费 ≈ 13,699元/天（降低当日NAV）
```

> **重要区分**：费用计提只是账面记录（降低NAV），**不是每天实际划款**。实际的费用划付通常按月或按季度执行。这两个逻辑在系统设计上必须分开处理。

### 5.9 现金管理类产品的特殊规定

根据银保监发〔2021〕20号，与TA系统直接相关的重要规定：

- **份额权益时间**：当日认购的份额自**下一个交易日**起享有分配权益；当日赎回的份额自下一个交易日起不享有分配权益
- **快速赎回上限**：单个投资者在单个销售渠道持有的单只产品单个自然日的赎回金额上限不高于**1万元**（即时到账），超过部分次日到账
- **规模约束**：同一理财公司采用摊余成本法的现管产品月末规模，合计不得超过其风险准备金月末余额的**200倍**

## 六、完整业务流程拆解

### 6.1 每日估值完整流程

```
【资管产品每日估值全流程】

  当日交易收盘（如下午3点）
        │
        ▼
  交易系统/前台系统
  · 锁定当日最终持仓数据，输出持仓快照
  · 推送给估值系统
        │
        ▼
  行情数据获取
  · 债券：从中债估值获取当日估值价格（收盘后约1-2小时内发布）
  · 股票：从交易所获取收盘价
  · 非标资产：使用约定的估值方法（成本法/独立评估）
        │
        ▼
  估值系统（核心计算）
  · 逐笔资产定价（市值法或摊余成本法）
  · 计算应计利息
  · 计提当日费用（管理费/托管费/销售服务费）
  · 计算当日NAV = （总资产 - 总负债）÷ 总份额
  · 现金管理类产品额外：计算影子定价NAV，计算偏离度，超限告警
  · 生成估值报告
        │
        ▼
  内部复核（自动校验+人工抽查）
        │
        ▼
  推送托管行独立复核
        │
        ▼
  托管行复核
  · 差异 ≤ 允许阈值 → 出具确认函，NAV正式生效
  · 差异 > 允许阈值 → 双方追溯原因，重新确认
        │
        ├─────────────────────┐
        ▼                     ▼
  推送TA系统              推送信息披露系统
  （用于份额清算）         （用于净值公告）
```

### 6.2 日终清算三条线

```
【日终清算三条线】

  ┌──────────────────────────────────────────────────┐
  │  资金清算（托管行主导）                              │
  │  · 申购资金归集确认（理财子：过渡户→托管户）          │
  │  · 赎回资金划付确认（托管户→投资者账户）             │
  │  · 投资交易资金交收（债券买卖的资金结算）             │
  └──────────────────────────────────────────────────┘
  ┌──────────────────────────────────────────────────┐
  │  份额清算（TA系统主导）                              │
  │  · 当日申购份额登记（投资者账户份额增加）             │
  │  · 当日赎回份额注销（投资者账户份额减少）             │
  │  · 产品总份额更新                                   │
  └──────────────────────────────────────────────────┘
  ┌──────────────────────────────────────────────────┐
  │  费用清算（估值系统计提 + 托管行定期划付）            │
  │  · 管理费/托管费/销售服务费/客户维护费 日计提         │
  │  · 注意：每日计提 ≠ 每日划款                         │
  │    计提是账面动作（降低NAV），实际划付按月/季度执行    │
  └──────────────────────────────────────────────────┘
```

### 6.3 托管行复核NAV的逻辑

```
【托管行复核NAV流程】

  管理人推送估值报告（持仓数据+计算过程+NAV结果）
        │
        ▼
  托管行用自己保存的持仓数据、行情数据、合同费率
  独立重新计算NAV
        │
  比对管理人NAV vs 托管行独立计算NAV
        │
  差异 ≤ 阈值          差异 > 阈值
        │                    │
        ▼                    ▼
  出具确认函           双方逐笔排查差异原因
  NAV正式生效          （持仓/行情/费率/利息计算？）
                       确认原因后重新计算确认
```

### 8.3 非标资产估值的特殊处理

| 问题维度 | 标准债券 | 非标资产 |
|----------|---------|---------|
| 估值数据来源 | 中债登每日发布 | 独立估值机构定期提供，或按成本法 |
| 估值频率 | 每日 | 月度/季度，非每日更新 |
| 估值方法 | 市值法（收益率曲线） | 成本法或现金流折现法 |
| 托管行复核难度 | 较低（有权威数据源） | 较高（缺乏公开市场价格参考） |

## 九、站在科技 / 产品 / 项目经理视角应该关注什么

### 9.1 估值方法配置是"业务规则"，不是"系统参数"

必须逐笔资产确认适用的估值方法，确认费率参数与产品说明书完全一致，确认现金管理类产品是否配置了影子定价机制。任何一个参数配错，都会导致NAV计算错误。

### 9.2 偏离度监控系统的设计要求

对于现金管理类产品，必须：
- 每日自动计算影子定价NAV和偏离度
- 设置分级告警（正偏离0.5%→暂停认购；负偏离0.25%→预警；负偏离0.5%→升级；连续2日超0.5%→触发紧急处置）
- 记录偏离度历史数据备监管查阅
- 偏离度超限时有自动业务拦截（如自动暂停认购）

### 9.3 系统建设优先级

| 优先级 | 功能模块 | 理由 |
|--------|----------|------|
| P0 | 标准债券摊余成本法计算 | 理财子主要产品类型核心需求 |
| P0 | 中债估值数据接入 | 市值法唯一权威数据源 |
| P0 | 费用计提（管理费/托管费） | 每日必须执行，影响NAV准确性 |
| P0 | 托管行数据推送与复核对接 | 合规强制要求 |
| P0 | 影子定价+偏离度监控 | 现金管理类产品监管强制要求 |
| P1 | 非标资产成本法处理 | 理财子特有需求 |
| P2 | 混合估值产品的灵活配置 | 复杂场景，可逐步建设 |

## 十一、常见误区与易混点

### 误区1：摊余成本法比市值法"落后"
不是落后，是不同的适用场景。资管新规限制的是滥用，不是消灭摊余成本法。

### 误区2：NAV由管理人单方面决定
NAV必须经过托管行独立复核确认后才能正式生效。管理人单方面发布的NAV是不合规的。

### 误区3：费用计提就是每天把钱划走
费用计提只是账面记录（降低NAV），不是每天实际划款。实际划付通常按月或按季度执行。

### 误区4：影子定价就是"市值法NAV"
影子定价是为现金管理类产品专门设计的风控工具，用途是"给摊余成本法做参照校验"，而不是替代摊余成本法作为日常申赎定价基础。

### 误区5：偏离度正负阈值对称
不对称。负偏离0.25%就要开始处置，比正偏离（0.5%才触发）更严格。这体现了监管对保护投资者利益的优先考量。

## 十二、案例分析

### 案例：某理财子"摊余转市值"改造引发的产品净值波动事件

**背景：**

某理财子旗下规模约30亿元的固收类理财产品，因持有的部分债券不再满足摊余成本法适用条件（开放频率从每季度改为每月），需要将这部分债券从摊余成本法切换为市值法。

**切换过程发生了什么：**

```
切换前（摊余成本法）：该债券账面价值 = 100.3元
切换当日（中债估值）：市场价格 = 98.5元（利率走高，债价下跌）
账面价值差异：100.3 - 98.5 = 1.8元/张

该债券占比20%，即6亿元持仓
账面价值下跌影响 ≈ -1,076万元

当日NAV变化：
  原NAV = 1.0500
  切换后NAV ≈ 1.0464（下跌约0.36%）
```

**投资者反应：** 净值单日下跌0.36%，远超日常波动幅度，引发部分投资者恐慌，当月开放日赎回量明显上升。

**教训：**

| 问题 | 教训 |
|------|------|
| 切换时机选择不当 | 应在市场利率低位（债券价格高）时切换，减少账面价值落差 |
| 未提前向投资者告知 | 估值方法重大变更应提前发布公告，做好预期管理 |
| 系统切换未经充分测试 | 需在测试环境完整演练后再上线 |
| 未提前与托管行沟通 | 重大参数变更应提前与托管行对齐 |

**（1）解释NAV计算流程：**
> "NAV不是一个简单的公式。每个交易日收盘后，估值系统要对每一笔底层资产重新定价——债券用中债估值，股票用交易所收盘价，非标用约定的方法——扣除当日计提的管理费、托管费等，才能算出当日净值。这个过程要到晚上才能完成，这也是为什么TA系统的份额确认要在夜间跑批。"

> "影子定价是现金管理类产品专用的风险控制工具——产品虽然用摊余成本法定价，但每天还要用市场价格重新算一遍影子净值，两者的差距就是偏离度。监管规定正偏离超0.5%要暂停认购，负偏离超0.25%就要开始处置，超0.5%连续两天还可能要终止产品清算。偏离度的阈值是不对称的，负偏离要求更严格。"

**（3）解释托管行复核的必要性：**
> "托管行要独立复核NAV，是因为管理人自己算的数不能自证清白。托管行是独立的第三方，它有产品的所有持仓记录，可以用自己的数据独立算一遍。两边对上了，NAV才能正式确认并对外披露。"

## 十九、我应该记住的关键句

**① `[业务理解]`**
> "NAV的计算不是一个简单的公式。每个交易日收盘后，估值系统要对每一笔底层资产重新定价——债券用中债估值，股票用交易所收盘价，非标用约定的方法——扣除当日的管理费、托管费计提，才能算出当日净值。这个过程需要几个小时，这就是为什么TA系统的份额确认要在夜间批处理。"

**② `[业务理解]`**
> "摊余成本法和市值法的本质区别是：摊余按买入成本加每日均匀摊销，净值曲线很平滑；市值法按当日市场价格重新定价，净值随市场波动。资管新规限制的不是摊余本身，而是用摊余来掩盖可以随时出售的资产的市场风险。"

**③ `[业务理解]`**
> "影子定价不是市值法的另一个名字，它是专门为现金管理类产品设计的风险控制工具。产品用摊余成本法定价，但每天还要用市场价格重新算一遍影子净值，两者之差除以摊余NAV就是偏离度。偏离度的阈值是不对称的：负偏离0.25%就要开始处置，正偏离要到0.5%才暂停认购。"

**④ `[系统表达]`**
> "托管行独立复核NAV，是资管合规体系的核心制衡机制。管理人算的数不能自证清白，托管行用自己保存的持仓数据和行情数据独立算一遍，两边对上才能确认。NAV必须经托管行确认后才能对外披露，这是有法律效力的程序。"

**⑤ `[系统表达]`**
> "日终清算是三条并行的线：资金清算（钱的流向，托管行主导）、份额清算（份额增减，TA主导）、费用清算（管理费等计提加定期划付）。特别要注意：费用计提每天都在降低净值，但不是每天都把钱划走——计提是账面动作，实际划付通常按月季度执行，这两件事在系统设计上必须分开处理。"

**⑥ `[业务理解]`**
> "理财子估值比公募复杂，核心原因有三：第一，摊余和市值经常在同一个产品里混用；第二，非标资产没有每日市场价格，估值数据来自定期的独立评估机构；第三，现金管理类产品还需要配套影子定价和偏离度监控，这些对估值系统的设计要求非常高。"

**⑦ `[自我定位]`**
> "我在理财子的经历覆盖了TA系统和产品管理系统，现在结合估值清算的知识，我能把三个系统之间的数据链条讲清楚：产品管理系统推送估值方法和费率参数给估值系统，估值系统算出NAV后推送给TA系统，TA系统用NAV完成份额清算。这条数据链上的每一个环节，我都有直接的业务认知。"

**⑧ `[面试收尾]`**
> "资管产品的估值，是整个后台体系的地基——净值算错了，所有基于净值的申赎交易都是错的，所有基于净值的信息披露都是失真的。我在理财子的工作经历，让我深刻理解了为什么这个地基要建得这么严格——因为它错不起。"
