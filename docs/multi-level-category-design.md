# 多级分类详细设计

## 1. 目标

在不破坏现有阅读、搜索、slug 路由和移动端交互的前提下，让笔记支持多级分类。

核心目标：

- 文件系统目录结构即分类结构
- 列表页支持“当前节点下全部笔记”
- 搜索范围从“当前分类”升级为“当前节点”
- 正文 breadcrumb 支持多级路径
- 兼容现有标题规范、课程排序和 slug 生成规则

## 2. 约束

当前项目的主要限制：

- 索引脚本只扫描 `notes/` 下一级目录
- 前端状态只支持单值 `currentCategory`
- 分类栏是平铺列表，不是树
- 搜索范围逻辑建立在一级分类之上

因此不采用“一次性无限层文件管理器式 UI”，而采用：

- 数据层：完整树形结构
- 交互层：树形分类 + 当前节点子树过滤

## 3. 数据结构

### 3.1 categoryTree

`notes/index.json` 新增 `categoryTree`，表示完整的分类树：

```json
{
  "name": "面试学习",
  "key": "面试学习",
  "path": ["面试学习"],
  "pathLabel": "面试学习",
  "depth": 0,
  "directNoteCount": 2,
  "totalNoteCount": 9,
  "children": [
    {
      "name": "券商",
      "key": "面试学习/券商",
      "path": ["面试学习", "券商"],
      "pathLabel": "面试学习 / 券商",
      "depth": 1,
      "directNoteCount": 1,
      "totalNoteCount": 4,
      "children": []
    }
  ]
}
```

### 3.2 allNotes

索引顶层新增扁平化 `allNotes`，作为前端列表、搜索、上一篇/下一篇的统一数据源：

```json
{
  "title": "BOCI 问答题库：短版",
  "source_title": "BOCI_QA_Short",
  "order": null,
  "unit": null,
  "slug": "面试学习-券商-boci-boci-问答题库-短版",
  "filename": "BOCI_QA_Short.md",
  "file": "notes/面试学习/券商/BOCI/BOCI_QA_Short.md",
  "preview": "...",
  "category": "BOCI",
  "rootCategory": "面试学习",
  "categoryKey": "面试学习/券商/BOCI",
  "categoryPath": ["面试学习", "券商", "BOCI"],
  "categoryPathLabel": "面试学习 / 券商 / BOCI"
}
```

### 3.3 search-index

`notes/search-index.json` 保持扁平结构，但补充路径语义字段：

- `category`
- `categoryPath`
- `categoryPathLabel`

## 4. 目录扫描规则

### 4.1 递归扫描

- 从 `notes/` 下一级目录开始递归扫描
- 目录层级即分类层级
- 每个目录中的 `.md` 文件都视为当前节点的直属笔记
- 子目录递归构成 `children`

### 4.2 README.md

当前版本会将 `README.md` 同时作为：

- 当前目录下的一篇直属笔记
- 当前分类节点的导航页/入口页

前端行为约定：

- 分类切换后，如果当前节点存在 `README.md`，欢迎页会优先给出“打开本节点导航页”的入口
- 笔记列表顶部会为该 `README` 渲染一张单独的导航卡片
- 该 `README` 不会再和普通笔记卡片混在一起重复出现

这样既保留了它作为笔记的可搜索性和可链接性，也让它在信息架构上承担真正的分类入口角色。

## 5. slug 与标题

### 5.1 标题

继续沿用现有规则：

- `front matter.title`
- 正文第一个 H1
- 文件名

课程类分类仍按已有规则自动规范为：

- `第N讲：主题`
- `第N课：主题`
- `Day N：主题`

### 5.2 slug

slug 的默认生成从：

- `分类 + 标题`

升级为：

- `完整分类路径 + 标题`

这样可以避免不同子分类下同名笔记更容易冲突。

## 6. 前端状态设计

### 6.1 当前节点

旧状态：

- `currentCategory`

新状态：

- `currentCategoryKey`

它表示当前选中的分类节点，例如：

- `面试学习`
- `面试学习/券商`
- `面试学习/券商/BOCI`

### 6.2 展开状态

新增：

- `expandedCategoryKeys`

用于控制树形分类栏展开/折叠。

规则：

- 根节点默认展开
- 当前选中节点的祖先强制展开
- 通过箭头按钮手动展开/收起

### 6.3 过滤规则

列表页展示规则从“当前分类完全匹配”升级为“当前节点子树匹配”：

- 未选节点：显示全部笔记
- 选中节点：显示 `categoryKey` 以该节点为前缀的全部笔记

## 7. UI 设计

### 7.1 桌面端

左栏改为树形导航：

- 行点击：选中当前节点并过滤列表
- 箭头点击：展开/收起子节点
- 计数显示：该节点子树总笔记数

### 7.2 移动端

移动端保持现有三栏结构，不引入新的复杂树面板。

分类栏仍然是左侧面板，但分类项改为树形缩进展示。

### 7.3 搜索

搜索范围文案从：

- 当前分类

升级为：

- 当前节点

语义上更准确。

## 8. 迁移策略

本次改造同步更新：

- `generate-index.py`
- `index.json`
- 前端分类树、搜索范围、列表过滤、breadcrumb

旧路由 `#note/<slug>` 和旧文件路径哈希继续兼容。

## 9. 非目标

本次不做：

- 分类说明页专用模板
- 拖拽排序
- 目录级权限
- 标签系统
- 收藏/置顶分类

## 10. 验证清单

- 一级分类仍可正常工作
- 多级目录中的笔记可显示在正确子树下
- 搜索“当前节点”只搜索当前子树
- 正文 breadcrumb 显示完整路径
- 上一篇/下一篇在当前节点上下文下仍正确
- slug 路由仍可打开笔记
