# 开发课程整合说明

## 背景

`/Users/shifusong/Projects/找工作/面试学习/开发` 不是零散笔记，而是一套完整的课程资料：

- 有总纲
- 有知识库首页
- 有模块 README
- 有大量 Markdown 内链

因此整合到站内时，目标不是“平铺所有文件”，而是保留课程结构，让它在网站里仍然像一套可学习的课程。

## 最终落位

课程整合到：

```text
notes/
  面试学习/
    开发岗/
      开发课程/
        README.md
        00_课程总纲/
        01_岗位与计划/
        02_系统与交付基础/
        03_项目表达/
        04_语言与数据基础/
        05_岗位定制补充/
      工银高盛/
```

说明：

- `开发课程` 负责系统学习
- `工银高盛` 保留为岗位/项目定制资料
- 两者都在 `面试学习 / 开发岗` 下，但职责不同

## 迁移原则

1. 不改动原始目录文件
2. 复制 Markdown 到 `notes/`
3. 排除不适合进站的工作记忆文件，例如 `_codex_memory.md`
4. 保留 `README.md`，把它当作课程入口页
5. 清洗本机绝对路径链接，避免站内出现死链

## 链接处理

导入时分三类处理：

1. 指向课程内部其他 Markdown 的链接  
   改写为站内笔记路径，前端再自动解析成站内跳转

2. 指向项目中已存在笔记的历史绝对路径链接  
   前端运行时兼容映射

3. 指向站外或本机私有文件的链接，例如 `.typ`、`.txt`  
   降级为普通说明文字，例如：

```text
当前开发简历（外部文件：sfs_zh 开发工程师.typ，站内未导入）
```

## slug 策略

课程类资料的 slug 改为“路径优先”：

- 普通笔记：目录路径 + 文件名
- `README.md`：目录路径

这样做的原因：

- 标题可以继续优化，不会让 URL 跟着飘
- 课程导航页天然有更干净的地址
- 多级分类与课程体系更稳定

同时保留历史标题型 slug 的兼容能力，避免旧链接立即失效。

## 使用建议

以后如果还有新的课程型资料：

1. 优先保留原始课程目录结构
2. 把 `README.md` 一并带入
3. 导入后运行：

```bash
cd /Users/shifusong/notes-site
python3 scripts/generate-index.py
```

4. 本地检查：
   - 分类树是否正确
   - README 是否排在当前目录前面
   - 站内内链是否能跳转
   - 搜索是否覆盖新课程
